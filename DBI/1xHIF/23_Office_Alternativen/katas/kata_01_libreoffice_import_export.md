# Kata 1: LibreOffice Import/Export

## Ziel

Lerne, wie man ein DOCX-Dokument in LibreOffice Writer öffnet, als ODT speichert, die Formatierung prüft und nach PDF exportiert.

---

## Aufgabenstellung

### Schritt 1: DOCX-Dokument erstellen

Erstelle in **Microsoft Word** ein Dokument mit folgendem Inhalt:

**Titel:** "Mein Testdokument" (Überschrift 1)

**Abschnitt 1: Einführung** (Überschrift 2)
Ein normaler Textabsatz zur Einführung des Themas.

**Abschnitt 2: Tabelle** (Überschrift 2)
Eine kleine Tabelle:

| Produkt | Preis | Anzahl |
|---------|-------|--------|
| Apfel | 1,20 € | 5 |
| Birne | 1,50 € | 3 |
| Banane | 0,90 € | 8 |

**Abschnitt 3: Bild** (Überschrift 2)
Füge ein beliebiges Bild ein (ClipArt oder eigenes Foto).

Speichere das Dokument als **testdokument.docx**.

---

### Schritt 2: In LibreOffice öffnen

1. Starte **LibreOffice Writer**
2. Gehe auf **Datei → Öffnen**
3. Wähle deine **testdokument.docx** Datei aus
4. Beobachte: Wie sieht die Formatierung aus?

**Prüfe:**
- Sind die Überschriften korrekt formatiert?
- Sieht die Tabelle gleich aus?
- Ist das Bild an der richtigen Position?
- Gibt es Unterschiede in der Schriftart?

**Notiere** alle Unterschiede, die dir auffallen.

---

### Schritt 3: Als ODT speichern

1. Gehe auf **Datei → Speichern unter**
2. Wähle als Dateityp: **ODF Textdokument (.odt)**
3. Dateiname: **testdokument.odt**
4. Klicke auf **Speichern**

Herzlichen Glückwunsch! Du hast dein erstes ODT-Dokument erstellt.

---

### Schritt 4: Formatierung vergleichen

Öffne **testdokument.odt** erneut (schließe es vorher kurz).

Vergleiche mit dem Original-DOCX:

| Element | DOCX | ODT (LibreOffice) | Unterschied? |
|---------|------|-------------------|--------------|
| Überschrift 1 | | | |
| Überschrift 2 | | | |
| Tabelle | | | |
| Bild | | | |
| Schriftart | | | |

Notiere die Unterschiede in einer Textdatei.

---

### Schritt 5: Nach PDF exportieren

1. Gehe auf **Datei → Exportieren als → Als PDF exportieren...**
2. Wähle die Standardeinstellungen
3. Speichere als **testdokument.pdf**

Ergebnis: Drei Dateien nebeneinander:
- `testdokument.docx` (Original)
- `testdokument.odt` (OpenDocument)
- `testdokument.pdf` (Portable)

---

## Hinweise

- LibreOffice verwendet standardmäßig andere Schriftarten als MS Office (z.B. Liberation Serif statt Times New Roman)
- Sehr komplexe Formatierungen (Textfelder, WordArt) werden nicht immer perfekt übernommen
- Die PDF-Exportfunktion von LibreOffice ist sehr gut — oft besser als die von Word
- Mit **Ansicht → Textfluss (Strg+Shift+W)** kannst du den Textfluss in LibreOffice umschalten

## Erweiterung (optional)

1. Erstelle ein DOCX mit nummerierten Listen, Fußnoten und einem Inhaltsverzeichnis
2. Teste, wie gut diese Elemente in LibreOffice übernommen werden
3. Öffne die ODT-Datei im **Editor/Notepad** — was siehst du? (Tipp: Es ist Zip+XML!)
