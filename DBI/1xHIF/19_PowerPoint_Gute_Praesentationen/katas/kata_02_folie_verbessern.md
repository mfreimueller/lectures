# Kata 2: Überladene Folie verbessern

## Ziel

Lerne "Less is More" anzuwenden, indem du eine stark überladene Folie auf das Wesentliche reduzierst und mit visuellen Elementen aufwertest.

## Aufgabenstellung

Die folgende Folie ist vollkommen überladen. Gestalte sie nach den gelernten Regeln neu.

---

### Ausgangssituation

Dein Mitschüler hat diese Folie für seinen Vortrag über "Cloud Computing" erstellt:

```
CLOUD COMPUTING - EINFUHRUNG

Cloud Computing ist ein Modell, das es ermöglicht, bei Bedarf von überall aus uber das
Internet auf gemeinsam nutzbare Rechenressourcen (wie Netzwerke, Server, Speicher,
Anwendungen und Dienste) zuzugreifen, die schnell bereitgestellt und wieder freigegeben
werden können, ohne dass ein direkter Verwaltungsaufwand durch den Nutzer entsteht.

Es gibt drei Haupt-Service-Modelle des Cloud Computings:
1. Infrastructure as a Service (IaaS): Der Anbieter stellt virtualisierte Rechenressourcen
   wie Server, Speicher und Netzwerke uber das Internet bereit. Der Kunde kann eigene
   Betriebssysteme und Anwendungen installieren und verwalten. Beispiele: AWS EC2,
   Google Compute Engine, Microsoft Azure Virtual Machines.
2. Platform as a Service (PaaS): Der Anbieter stellt eine Plattform zur Verfügung, auf der
   Kunden eigene Anwendungen entwickeln, testen und bereitstellen können, ohne sich um
   die zugrundeliegende Infrastruktur kümmern zu müssen. Beispiele: Google App Engine,
   Heroku, Microsoft Azure App Services.
3. Software as a Service (SaaS): Der Anbieter stellt vollständige Softwareanwendungen uber
   das Internet bereit. Der Kunde nutzt die Software ohne Installation oder Wartung.
   Beispiele: Google Drive, Microsoft 365, Dropbox, Salesforce.

Vorteile von Cloud Computing:
- Skalierbarkeit: Ressourcen können bei Bedarf erhöht oder verringert werden
- Kosteneffizienz: Nur für tatsächlich genutzte Ressourcen wird bezahlt (Pay-as-you-go)
- Flexibilität: Von überall aus zugänglich (ortsunabhängiges Arbeiten)
- Automatische Updates: Der Anbieter kümmert sich um Wartung und Sicherheitsupdates
- Zuverlässigkeit: Hohe Verfügbarkeit durch redundante Systeme (99,9% uptime)

Nachteile von Cloud Computing:
- Abhängigkeit vom Internet (ohne Verbindung kein Zugriff)
- Datenschutzbedenken (Daten liegen auf fremden Servern)
- Vendor Lock-in (Abhängigkeit von einem Anbieter)
- Sicherheitsrisiken (potenzielle Angriffspunkte)

Quelle: https://de.wikipedia.org/wiki/Cloud_Computing (letzter Zugriff: 15.03.2026)
```

---

### Aufgaben

1. **Analysieren:** Welche Fehler hat diese Folie? Notiere 3-5 konkrete Probleme.
2. **Reduzieren:** Reduziere den Inhalt auf die wichtigsten Kernaussagen. Was muss wirklich auf die Folie?
3. **Strukturieren:** Teile den Inhalt auf mehrere Folien auf (max. 3 Folien).
4. **Visualisieren:** Ergänze passende visuelle Elemente (Symbole, Icons, ein Bild pro Folie).
5. **Gestalten:** Wähle passende Farben, Schriftgrößen und ein einheitliches Layout.

---

## Hinweise

**Wende diese Regeln an:**
- **Less is More:** Maximal 6-8 Zeilen pro Folie, 6-8 Wörter pro Zeile
- **Schriftgröße:** Mindestens 24-30 pt, Überschrift 36-44 pt
- **Aufteilung:** Eine Kernaussage pro Folie
- **Struktur:** Überschrift -> max. 4-5 Bullet Points
- **Visualisierung:** Jede Folie sollte ein Bild, Diagramm oder Icon enthalten

**Vorschlag zur Aufteilung:**
- **Folie 1:** Definition Cloud Computing (1 Satz + passendes Bild)
- **Folie 2:** Service-Modelle (IaaS, PaaS, SaaS) mit kurzer Erklärung
- **Folie 3:** Vor- und Nachteile (je 2-3 Stichworte + Bild)

**Tipp:** Zeichne zuerst einen groben Entwurf auf Papier, bevor du die Folie in PowerPoint umsetzt.
