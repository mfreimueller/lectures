Working with Databases Using Java

Connecting Programming and Data Storage

Presenter: Michael Freimüller

Class: 2EHIF

Date: March 26, 2026

Why Combine Databases and Programming?

Without Databases:

  • Data stored in memory only
  • Loses everything on program end
  • No data persistence between users

With Databases:

  • Data persists after program ends
  • Multiple users can access data
  • Reliable and organized storage

Learning Objectives

After this lesson, you will be able to:

  1. Connect Java applications to MSSQL and MariaDB databases
  2. Write both regular and prepared SQL statements in Java
  3. Understand when to use each type of statement
  4. Explain what SQL injection is and how to prevent it
  5. Apply secure database practices in your own projects

Database Connection Overview

To connect Java to a database, we need:

Step 1: JDBC Driver

Add the database driver to your project (Maven/Gradle or manual JAR)

Step 2: Connection String

Specify database type, host, port, and name

Step 3: Credentials

Username and password for authentication

Step 4: Connection Object

Establish the actual connection using DriverManager

MSSQL Database Connection

Maven Dependency (pom.xml):


<dependency>
    <groupId>com.microsoft.sqlserver</groupId>
    <artifactId>mssql-jdbc</artifactId>
    <version>13.4.0.jre11</version>
    <scope>compile</scope>
</dependency>

Java Connection Code:

String url = "jdbc:sqlserver://localhost:1433;databaseName=SchulDb;trustServerCertificate=true";
String user = "sa";
String password = "your_password";

Connection connection = DriverManager.getConnection(url, user, password);

MariaDB Database Connection

Maven Dependency (pom.xml):

<dependency>
    <groupId>org.mariadb.jdbc</groupId>
    <artifactId>mariadb-java-client</artifactId>
    <version>3.1.4</version>
    <scope>compile</scope>
</dependency>

Java Connection Code:

String url = "jdbc:mariadb://localhost:3306/mydb";
String user = "root";
String password = "your_password";

Connection connection = DriverManager.getConnection(url, user, password);

Regular SQL Statements

Use Statement for simple, non-dynamic queries:

// Create a Statement object
Statement stmt = connection.createStatement();

// Execute a SELECT query
String sql = "SELECT * FROM students WHERE id = 1";
ResultSet rs = stmt.executeQuery(sql);

// Process results 
while(rs.next()) {
    String name = rs.getString("name");
    int age = rs.getInt("age");
    System.out.println(name + ", Age: " + age);
}

Prepared Statements

What are they?

  • Pre-compiled SQL queries with placeholders (?)
  • Parameters are safely inserted by the database driver
  • Cannot be manipulated by malicious input

Why use them?

  1. Security (prevents SQL injection)
  2. Better performance for repeated queries
  3. Cleaner code when handling user input

When to Use Each Type

Use Statement When:

  • Query is fixed and hardcoded
  • No user input involved
  • Simple administrative queries

Use PreparedStatement When:

  • Query contains user input
  • Form submissions or API calls
  • Any dynamic WHERE clauses

SQL Injection Attack

Vulnerable Code (DO NOT USE):

String userInput = "' OR '1'='1";
String sql = "SELECT * FROM users WHERE username = '" + userInput + "'";
// Result: SELECT * FROM users WHERE username = '' OR '1'='1' --

What happens?

  • The query returns ALL users
  • Authentication can be bypassed completely
  • Attacker can delete or modify data

Vulnerable vs Secure Code

DANGEROUS (Statement):

String name = userInput;
String sql = "SELECT * FROM users WHERE name = '" + name + "'";
stmt.executeQuery(sql);

SECURE (PreparedStatement):

String name = userInput;
String sql = "SELECT * FROM users WHERE name = ?";
PreparedStatement ps = connection.prepareStatement(sql);
ps.setString(1, name);
ps.executeQuery();

Syntax Breakdown

Complete example structure:

// 1. Establish connection
Connection conn = DriverManager.getConnection(url, user, pass);

// 2. Create PreparedStatement
String sql = "INSERT INTO students (name, age) VALUES (?, ?)";
PreparedStatement ps = conn.prepareStatement(sql);

// 3. Set parameters (indices start at 1!)
ps.setString(1, "John Doe");
ps.setInt(2, 17);

// 4. Execute
int rowsAffected = ps.executeUpdate();
System.out.println(rowsAffected + " row(s) inserted");

Summary - Key Takeaways

  • JDBC Driver: Required to connect Java to any database (MSSQL, MariaDB, PostgreSQL, etc.)
  • Connection String: Format is jdbc:[database_type]://host:port/database_name
  • Statement vs PreparedStatement: Use PreparedStatement for ALL user input!
  • SQL Injection: A serious security vulnerability that can be completely prevented with prepared statements
  • Error Handling: Always use try-with-resources to properly close database connections

Assessment Questions

Question 1:

What is the main security advantage of PreparedStatement over Statement?

Answer: It prevents SQL injection by treating user input as data, not executable code

Question 2:

Why should you use try-with-resources when working with database connections?

Answer: It automatically closes connections and prevents resource leaks
