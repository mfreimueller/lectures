Wiederholung: if-Anweisungen (Woche 3)

Syntax einer einfachen if-Bedingung:

if (bedingung) {
    // Code wird ausgeführt, wenn Bedingung wahr ist
}

Wiederholung: if-Anweisungen (Woche 3)

Beispiel: Altersprüfung

int alter = 18;

if (alter >= 18) {
    System.out.println("Zugang erlaubt");
}

Verschachtelte Verzweigungen

Eine if-Bedingung kann selbst wieder eine weitere if-Abfrage enthalten:

if (alter >= 18) {
    System.out.println("Erwachsen");
    
    if (kreditkarteVorhanden) {
        System.out.println("Kauf möglich");
    } else {
        System.out.println("Nur Barzahlung");
    }
} else {
    System.out.println"Minderjährig");
}

if-else-if-Kaskaden

Mehrere Bedingungen nacheinander prüfen:

if (note >= 90) {
    System.out.println("Sehr gut");
} else if (note >= 80) {
    System.out.println("Gut");
} else if (note >= 70) {
    System.out.println("Befriedigend");
} else if (note >= 60) {
    System.out.println("Genügend");
} else {
    System.out.println("Nicht genügend");
}
Hinweis: Die Bedingungen werden von oben nach unten abgearbeitet. Sobald eine Bedingung wahr ist, wird der zugehörige Block ausgeführt und die restlichen Bedingungen übersprungen.

Logische Operatoren

&& (UND): Beide Bedingungen müssen wahr sein

alter >= 18 && passwortRichtig

|| (ODER): Mindestens eine Bedingung muss wahr sein

alter >= 6 || alter >= 12

Logische Operatoren

! (NICHT): Bedingung umkehren

!(alter < 18) // bedeutet: alter >= 18

Praxisbeispiel: Login-System

Zwei Bedingungen müssen gleichzeitig erfüllt sein:

String benutzername = "admin";
String passwort = "1234";

if (benutzername.equals("admin") && 
    passwort.equals("1234")) {
    System.out.println("Zugriff gewährt");
} else {
    System.out.println("Zugriff verweigert");
}

Betrachtet folgenden Code

int punkte = 75;
boolean quizBestanden = true;

if (punkte >= 50 && quizBestanden) {
    System.out.println("Kurs bestanden");
} else if (punkte >= 50 || !quizBestanden) {
    System.out.println("Teilbestanden");
} else {
    System.out.println("Nicht bestanden");
}

Häufige Fehlerquellen

Fehler 1: = statt == verwenden

// FALSCH: if (x = 5) { ... }
// RICHTIG: if (x == 5) { ... }

Häufige Fehlerquellen

// FALSCH:
if (x > 5)
    System.out.println("x > 5");
    System.out.println("Immer");

Übung 1: Notenberechnung

Aufgabe:

  • Erstelle ein Programm, das eine Bewertung (0-100) als Eingabe erhält
  • Die entsprechende Schulnote wird ausgegeben

Übung 1: Notenberechnung

Anforderungen:

90-100 = "Sehr gut"
80-89  = "Gut"
70-79  = "Befriedigend"
60-69  = "Genügend"
0-59   = "Nicht genügend"

Vorwissen: Vergleichsoperatoren, if-Anweisungen

Übung 2: Login-System (einfache Version)

Aufgabe:

  • Benutzername und Passwort abfragen
  • Richtige Eingabe: "Zugriff gewährt"
  • Falsche Eingabe: "Zugriff verweigert"

Übung 2: Login-System (einfache Version)

Vorgaben:

Benutzername: "admin"
Passwort: "1234"

Tipp: Verwende den logischen Operator && um beide Bedingungen zu verknüpfen

Übung 3: Kleiner Taschenrechner

Aufgabe:

  • Benutzer wählt Rechenoperation (+, -, *, /)
  • Zwei Zahlen eingeben lassen
  • Ergebnis berechnen und ausgeben

Übung 3: Kleiner Taschenrechner

Anforderungen:

1 = Addition (+)
2 = Subtraktion (-)
3 = Multiplikation (*)
4 = Division (/)
Hinweis: Divisionen durch 0 dürfen nicht erlaubt sein!

Übung 4: Altersfreigabe

Szenario:

  • Film hat FSK-Einstufung
  • Programm prüft Alter des Zuschauers
  • FSK-16: mind. 18-jährige Begleitung erforderlich

Übung 4: Altersfreigabe

Anforderungen:

FSK 6: Ab 6 Jahren allein möglich
FSK 12: Ab 12 Jahren allein möglich
FSK 16: Ab 16 oder mit mind. 18-jähriger Begleitung
FSK 18: Nur ab 18 Jahren

Dies erfordert verschachtelte Verzweigungen und logische Operatoren!

Zusammenfassung

  • Verschachtelte if-else-Anweisungen ermöglichen komplexe Entscheidungsstrukturen
  • && (UND): Beide Bedingungen müssen wahr sein
  • || (ODER): Mindestens eine Bedingung muss wahr sein
  • ! (NICHT): Kehrt den Wahrheitswert um

Zusammenfassung

Achtung bei mehreren if-else Bedingungen:

if (bedingung1) { ... }
else if (bedingung2) { ... }
else { ... }

Jede Bedingung wird nur geprüft, wenn vorherige Bedingungen falsch waren!
