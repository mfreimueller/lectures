Warum sortieren?

  • Suchen: sortierte Daten sind schneller durchsuchbar (Binäre Suche)
  • Darstellen: geordnete Ausgabe (Highscores, Ranglisten)
  • Rankings: der Beste/Erste steht ganz vorne
  • Datenanalyse: Muster erkennen in sortierten Daten

Selection Sort: Idee

Das kleinste Element suchen und an den Anfang stellen.

  1. Suche das Minimum im gesamten Array
  2. Tausche es mit dem Element an Position 0
  3. Suche das Minimum ab Position 1
  4. Tausche es mit dem Element an Position 1
  5. Wiederhole, bis das Array sortiert ist

Selection Sort: Visualisierung

// Start: [5, 2, 8, 1, 9]
// Schritt 1: Minimum = 1 an Index 3 -> tausche mit Index 0
//           [1, 2, 8, 5, 9]
// Schritt 2: Minimum ab Index 1 = 2 (bereits richtig)
//           [1, 2, 8, 5, 9]
// Schritt 3: Minimum ab Index 2 = 5 an Index 3 -> tausche mit Index 2
//           [1, 2, 5, 8, 9]
// Schritt 4: Minimum ab Index 3 = 8 (bereits richtig)
//           [1, 2, 5, 8, 9]

Selection Sort: Code

void selectionSort(int[] arr) {
    for (int i = 0; i < arr.length - 1; i++) {
        // Minimum ab Position i suchen
        int minIndex = i;
        for (int j = i + 1; j < arr.length; j++) {
            if (arr[j] < arr[minIndex]) {
                minIndex = j;
            }
        }
        // Minimum an Position i tauschen
        int temp = arr[i];
        arr[i] = arr[minIndex];
        arr[minIndex] = temp;
    }
}

Live-Coding: Selection Sort

int[] zahlen = {5, 2, 8, 1, 9};
selectionSort(zahlen);
System.out.println(Arrays.toString(zahlen));
// [1, 2, 5, 8, 9]

Insertion Sort: Idee

Elemente einordnen wie Spielkarten:

  1. Beginne mit dem zweiten Element
  2. Vergleiche es mit den Elementen davor
  3. Verschiebe grössere Elemente nach rechts
  4. Füge das aktuelle Element an der richtigen Stelle ein
  5. Wiederhole bis zum Ende des Arrays

Insertion Sort: Visualisierung

// Start: [5, 2, 8, 1, 9]
// Schritt 1: 2 mit 5 vergleichen, 5 nach rechts
//           [2, 5, 8, 1, 9]
// Schritt 2: 8 mit 5 vergleichen, bleibt
//           [2, 5, 8, 1, 9]
// Schritt 3: 1 mit 8,5,2 vergleichen, alle nach rechts
//           [1, 2, 5, 8, 9]
// Schritt 4: 9 mit 8 vergleichen, bleibt
//           [1, 2, 5, 8, 9]

Insertion Sort: Code

void insertionSort(int[] arr) {
    for (int i = 1; i < arr.length; i++) {
        int aktuell = arr[i];
        int j = i - 1;

        // Grössere Elemente nach rechts verschieben
        while (j >= 0 && arr[j] > aktuell) {
            arr[j + 1] = arr[j];
            j--;
        }

        // Aktuelles Element einfügen
        arr[j + 1] = aktuell;
    }
}

Live-Coding: Insertion Sort

int[] zahlen = {5, 2, 8, 1, 9};
insertionSort(zahlen);
System.out.println(Arrays.toString(zahlen));
// [1, 2, 5, 8, 9]

Vergleich: Selection vs. Insertion

KriteriumSelection SortInsertion Sort
IdeeMinimum suchen, tauschenElemente einordnen
Best-CaseO(n^2)O(n) (fast sortiert)
Average-CaseO(n^2)O(n^2)
StabilNeinJa
SpeicherIn-placeIn-place

Zeitkomplexität

  • Beide Algorithmen haben im Durchschnitt O(n^2)
  • Für n Elemente: ca. n^2/2 Vergleiche
  • Bei n=10: ca. 50 Vergleiche
  • Bei n=1000: ca. 500.000 Vergleiche
  • Insertion Sort ist bei fast sortierten Arrays schneller (O(n))

Dungeon Escape: Highscore sortiert

int[] scores = {85, 120, 45, 200, 70};
String[] namen = {"Aragorn", "Legolas", "Frodo", "Gandalf", "Gimli"};

// Parallel sortieren: scores und namen
for (int i = 0; i < scores.length - 1; i++) {
    int minIndex = i;
    for (int j = i + 1; j < scores.length; j++) {
        if (scores[j] < scores[minIndex]) {
            minIndex = j;
        }
    }
    int tempScore = scores[i];
    scores[i] = scores[minIndex];
    scores[minIndex] = tempScore;

    String tempName = namen[i];
    namen[i] = namen[minIndex];
    namen[minIndex] = tempName;
}

Dungeon Escape: Aufgabe

Implementiere eine Highscore-Liste für Dungeon Escape:

  • Zwei Arrays: int[] scores und String[] namen
  • Sortiere beide Arrays parallel mit Selection Sort
  • Gib die sortierte Highscore-Liste aus

Zusammenfassung

  • Selection Sort: Minimum suchen und an Position i tauschen
  • Insertion Sort: Elemente einordnen, grössere nach rechts verschieben
  • Beide Algorithmen: O(n^2) im Durchschnitt
  • Insertion Sort ist bei fast sortierten Arrays effizienter
  • Paralleles Sortieren: beide Arrays gleichzeitig umsortieren
