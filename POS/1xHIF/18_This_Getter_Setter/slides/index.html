18 - this, Getter und Setter

POS - 1AHIF

Wiederholung: Konstruktoren

public class Auto {
    String marke;
    int baujahr;

    public Auto(String marke, int baujahr) {
        this.marke = marke;
        this.baujahr = baujahr;
    }
}
  • this verweist auf das aktuelle Objekt
  • this.marke = Attribut, marke = Parameter
  • Heute: this genaür, plus Kapselung mit private

Problem: Attribut vs. Parameter

public class Auto {
    String marke; // Attribut

    // Ohne this:
    public Auto(String m, int b) {
        marke = m;  // funktioniert, aber unklar
    }

    // Mit this:
    public Auto(String marke, int baujahr) {
        this.marke = marke;  // klar!
        this.baujahr = baujahr;
    }
}
  • Ohne this: müssen verschiedene Namen erfinden (m statt marke)
  • Mit this: können gleiche Namen verwenden

this.attribut = parameter

this = "dieses Objekt" (die aktuelle Instanz)

public class Person {
    String name;

    public void setName(String name) {
        this.name = name;
        // this.name = Attribut (gehört zum Objekt)
        // name     = Parameter (wird übergeben)
    }
}
  • this.name ist das Attribut des Objekts
  • name (ohne this) ist der Parameter der Methode

Kapselung: private vs. public

  • public - von überall zugreifbar (auch aus anderen Klassen)
  • private - nur innerhalb derselben Klasse zugreifbar

Grundprinzip: Attribute private, Methoden public

public class Konto {
    private long kontostand; // nur innerhalb Konto sichtbar

    public long getKontostand() {
        return kontostand; // erlaubt (gleiche Klasse)
    }
}

Warum private Attribute?

  • Daten schützen: Niemand soll direkt fremde Attribute ändern können
  • Kontrolle: Nur die Klasse selbst entscheidet, wie Daten gelesen/geschätzt werden
  • Änderbarkeit: Interne Änderungen wirken sich nicht auf andere Klassen aus
  • Validierung: Ungültige Werte können verhindert werden
// OHNE private: jeder kann kontostand auf -1000 setzen
// MIT private: nur über Methoden mit Prüfung

Getter = lesender Zugriff

public class Konto {
    private long kontostand; // privat - kein direkter Zugriff

    // Getter: erlaubt das Lesen von aussen
    public long getKontostand() {
        return kontostand;
    }
}

// Verwendung:
Konto k = new Konto();
long stand = k.getKontostand(); // legal
// k.kontostand = 1000; // ILLEGAL (private)
  • Getter sind public - sie ermöglichen kontrollierten Lese-Zugriff
  • Namenskonvention: get + Attributname (Grossbuchstabe)

Setter = schreibender Zugriff

public class Konto {
    private long kontostand;

    // Setter: erlaubt kontrolliertes Schreiben von aussen
    public void setKontostand(long kontostand) {
        // mit Validierung!
        if (kontostand >= 0) {
            this.kontostand = kontostand;
        }
    }
}
  • Setter sind public void - sie erlauben kontrollierten Schreib-Zugriff
  • Hier kann Validierung eingebaut werden
  • Namenskonvention: set + Attributname (Grossbuchstabe)

Live-Coding: Gekapseltes Konto

public class Konto {
    private long kontostand;

    public long getKontostand() {
        return kontostand;
    }

    public void einzahlen(long betrag) {
        if (betrag > 0) {
            kontostand = kontostand + betrag;
        }
    }

    public void auszahlen(long betrag) {
        if (betrag > 0 && kontostand >= betrag) {
            kontostand = kontostand - betrag;
        }
    }
}

Setter mit Validierung

public class Konto {
    private long kontostand;

    // Setter mit Prüfung
    public void setKontostand(long kontostand) {
        if (kontostand < 0) {
            System.out.println("Negativer Betrag nicht erlaubt!");
            return; // Methode beenden, nichts ändern
        }
        this.kontostand = kontostand;
    }
}

// Verwendung:
konto.setKontostand(-100); // "Negativer Betrag nicht erlaubt!"
konto.setKontostand(500);  // OK
  • Prüfe eingehende Werte bevor sie gesetzt werden
  • Verhindere ungültige Zustände

Getter - nur lesen, nicht ändern

public class Konto {
    private long kontostand;

    // Getter: gibt den Wert zurück
    public long getKontostand() {
        return kontostand;
    }

    // KEIN Setter für kontostand - Änderungen nur über
    // einzahlen() und auszahlen() mit eigener Logik
    public void einzahlen(long betrag) { ... }
    public void auszahlen(long betrag) { ... }
}
  • Manchmal will man nur lesen aber nicht direkt schreiben
  • Kein Setter = das Attribut kann nicht von aussen gesetzt werden

Beispiel: Temperatur

public class Temperatur {
    private double celsius;

    public double getCelsius() {
        return celsius;
    }

    public void setCelsius(double celsius) {
        this.celsius = celsius;
    }

    // Berechneter Getter: Fahrenheit wird aus celsius berechnet
    public double getFahrenheit() {
        return celsius * 9.0 / 5.0 + 32.0;
    }
}
  • Nur celsius wird gespeichert
  • Fahrenheit wird berechnet - kein extra Attribut

Dungeon Escape: Kapselung

public class Spieler {
    private String name;
    private int hp;
    private int maxHp;
    private int angriff;

    // Getter
    public String getName() { return name; }
    public int getHp() { return hp; }
    public int getAngriff() { return angriff; }

    // Setter mit Prüfung
    public void setHp(int hp) {
        if (hp < 0) this.hp = 0;
        else if (hp > maxHp) this.hp = maxHp;
        else this.hp = hp;
    }
}

HP nie kleiner 0, nie grösser als maxHp!

Dungeon Escape Aufgabe

  • Mache alle Attribute private
  • Erstelle Getter für name, hp, angriff, verteidigung
  • Erstelle setHp(int) mit Prüfung: HP nie < 0 oder > maxHp
  • Konstruktor setzt name, hp=100, maxHp=100, angriff=15, verteidigung=8
Spieler held = new Spieler("Held");
held.setHp(50);  // OK
held.setHp(200); // wird auf maxHp (100) begrenzt
held.setHp(-10); // wird auf 0 gesetzt

Zusammenfassung

  • this - Verweis auf das aktuelle Objekt, unterscheidet Attribut von Parameter
  • private - Zugriff nur innerhalb der eigenen Klasse
  • public - Zugriff von überall
  • Getter - kontrollierter Lese-Zugriff (public int getX())
  • Setter - kontrollierter Schreib-Zugriff mit Validierung (public void setX(int x))
