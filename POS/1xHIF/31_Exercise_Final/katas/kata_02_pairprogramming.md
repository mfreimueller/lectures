# Kata 02: Pair-Programming - Code-Review

Analysiere den Dungeon Escape Code deines Partners und finde 3 Bugs oder Verbesserungsvorschläge.

## Aufgaben
1. Lies den Code deines Partners sorgfältig durch
2. Notiere mindestens 3 Probleme oder Verbesserungen
3. Besprecht eure Funde gemeinsam

## Mögliche Fehler
- Off-by-one Fehler bei Zufallszahlen
- Fehlende Grenzüberprüfungen
- Logikfehler in Kampfmechaniken
- Keine oder falsche JavaDoc
- Inkonsistente Namensgebung

## Dokumentation
Halte deine Funde in dieser Datei fest oder notiere sie als Kommentar im Code.
