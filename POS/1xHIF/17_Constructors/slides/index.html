17 - Konstruktoren

POS - 1AHIF

Wiederholung: new und Objekte

Auto a = new Auto();
a.marke = "VW";
a.baujahr = 2020;
a.fahre();
  • new Auto() erzeugt ein Objekt
  • Danach müssen wir Attribute manuell setzen
  • Das ist umständlich - besonders bei vielen Attributen

Das Problem

Drei Schritte für ein Objekt: erzeugen, Attribute setzen, verwenden

Auto a = new Auto();     // 1. Erzeugen
a.marke = "VW";          // 2. Attribute setzen
a.baujahr = 2020;        // 2. Attribute setzen
a.fahre();               // 3. Verwenden

Was, wenn wir vergessen, ein Attribut zu setzen?
Was, wenn wir 10 Attribute haben?

Lösung: Konstruktor - Initialisierung direkt bei der Erzeugung

Was ist ein Konstruktor?

  • Ein Konstruktor ist eine spezielle Methode zur Initialisierung
  • Er hat denselben Namen wie die Klasse
  • Er hat keinen Rückgabetyp (nicht einmal void)
  • Er wird automatisch bei new aufgerufen
public class Auto {
    String marke;
    int baujahr;

    // Konstruktor
    public Auto(String marke, int baujahr) {
        this.marke = marke;
        this.baujahr = baujahr;
    }
}

Standardkonstruktor

  • Wenn wir keinen Konstruktor definieren, erzeugt Java einen Standardkonstruktor
  • Der Standardkonstruktor hat keine Parameter
  • Er setzt Attribute auf Standardwerte (0, null, false)
// Von Java automatisch erzeugt:
public Auto() {
    // nichts - Attribute bleiben auf Standardwerten
}

Sobald wir irgendeinen Konstruktor definieren, gibt es keinen automatischen Standardkonstruktor mehr!

Parametrisierter Konstruktor

public class Auto {
    String marke;
    int baujahr;

    public Auto(String marke, int baujahr) {
        this.marke = marke;
        this.baujahr = baujahr;
    }
}
  • Übergibt Werte direkt bei der Erzeugung
  • this.marke = Attribut, marke = Parameter
  • Kein manuelles Setzen mehr nötig!

Syntax: Konstruktor

[Zugriffsmodifizierer] Klassenname(Parameter) {
    // Initialisierung
}
  • public - damit wir von überall darauf zugreifen können
  • Klassenname - muss exakt dem Klassennamen entsprechen
  • Parameter - Werte, die beim Erzeugen übergeben werden
  • this.attribut = parameter - Attribut mit Parameterwert setzen

Live-Coding: Auto mit Konstruktor

public class Auto {
    String marke;
    int baujahr;

    public Auto(String marke, int baujahr) {
        this.marke = marke;
        this.baujahr = baujahr;
    }

    void fahre() {
        System.out.println(marke + " fährt!");
    }
}

// Verwendung:
Auto a = new Auto("VW", 2020);
a.fahre();  // VW fährt!

Konstruktor-Überladung

  • Eine Klasse kann mehrere Konstruktoren haben
  • Sie unterscheiden sich in der Parameterliste (Anzahl oder Typ)
  • Java wählt automatisch den passenden Konstruktor aus
public class Auto {
    String marke;
    int baujahr;

    public Auto() {
        marke = "Unbekannt";
        baujahr = 0;
    }

    public Auto(String marke, int baujahr) {
        this.marke = marke;
        this.baujahr = baujahr;
    }
}

Beispiel: Person mit Überladung

public class Person {
    String name;
    int alter;

    // 1. Keine Parameter
    public Person() {
        name = "Unbekannt";
        alter = 0;
    }

    // 2. Nur Name
    public Person(String name) {
        this.name = name;
        this.alter = 0;
    }

    // 3. Name und Alter
    public Person(String name, int alter) {
        this.name = name;
        this.alter = alter;
    }
}

Alle drei Konstruktoren sind nutzbar: new Person(), new Person("Max"), new Person("Max", 25)

this(parameter) - Konstruktor ruft Konstruktor

public class Person {
    String name;
    int alter;

    public Person() {
        this("Unbekannt", 0); // ruft den 3. Konstruktor
    }

    public Person(String name) {
        this(name, 0);        // ruft den 3. Konstruktor
    }

    public Person(String name, int alter) {
        this.name = name;
        this.alter = alter;
    }
}
  • Vermeidet Code-Wiederholung
  • this() muss die erste Anweisung im Konstruktor sein

Dungeon Escape: Spieler-Konstruktor

public class Spieler {
    String name;
    int hp;
    int angriff;

    public Spieler(String name) {
        this.name = name;
        this.hp = 100;

Spieler held = new Spieler("Held");

Ein Konstruktor, der Standardwerte setzt - viel sauberer!

Dungeon Escape: Raum-Konstruktor

public class Raum {
    String name;
    String beschreibung;

    public Raum(String name, String beschreibung) {
        this.name = name;
        this.beschreibung = beschreibung;
    }
}

// Optional: Raum mit Gegner
public class Raum {
    String name;
    String beschreibung;
    String gegner;

    public Raum(String name, String beschreibung) {
        this(name, beschreibung, null);
    }

    public Raum(String name, String beschreibung, String gegner) {
        this.name = name;
        this.beschreibung = beschreibung;
        this.gegner = gegner;
    }
}

Zusammenfassung

  • Konstruktor = Initialisierung beim Objekt-Erzeugen
  • Gleicher Name wie die Klasse, kein Rückgabetyp
  • Standardkonstruktor - automatisch, wenn keiner definiert
  • Parametrisierter Konstruktor - Werte direkt übergeben
  • Überladung - mehrere Konstruktoren mit unterschiedlichen Parametern
  • this(parameter) - Konstruktor ruft anderen Konstruktor auf
