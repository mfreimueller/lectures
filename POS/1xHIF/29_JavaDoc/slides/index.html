Was ist JavaDoc?

  • Dokumentation aus speziellen Kommentaren im Code
  • Wird zu HTML-Seiten verarbeitet
  • Ermoeglicht professionelle API-Dokumentation
  • Beispiel: java.lang.String-Dokumentation von Oracle

Syntax: Kommentararten

ArtSyntaxVerwendung
Einzeiler// textKurze Notizen
Block/* text */Mehrzeilige Notizen
JavaDoc/** text */Dokumentation

JavaDoc-Tags

TagVerwendung
@paramBeschreibt einen Methodenparameter
@returnBeschreibt den Rueckgabewert
@authorAutor der Klasse
@versionVersionsnummer
@seeVerweis auf andere Klassen/Methoden

Klassen-JavaDoc

Wird oberhalb der Klasse geschrieben:

/**
 * Diese Klasse repraesentiert ein Bankkonto.
 *
 * @author Max Mustermann
 * @version 1.0
 */
class Bankkonto {
    ...
}

Methoden-JavaDoc

Wird oberhalb der Methode geschrieben:

/**
 * Zahlt einen Betrag auf das Konto ein.
 *
 * @param betrag Der einzuzahlende Betrag
 */
public void einzahlen(double betrag) {
    kontostand += betrag;
}

Live-Coding: Bankkonto dokumentieren

Eine Klasse Bankkonto mit JavaDoc versehen:

/**
 * Bankkonto mit Ein- und Auszahlungen.
 * @author Max Mustermann
 * @version 1.0
 */
class Bankkonto {
    private double kontostand;

    /**
     * Zahlt einen Betrag ein.
     * @param betrag Einzahlungsbetrag
     */
    public void einzahlen(double betrag) {
        kontostand += betrag;
    }

    /**
     * Gibt den Kontostand zurueck.
     * @return aktueller Kontostand
     */
    public double getKontostand() {
        return kontostand;
    }
}

Gute vs. schlechte JavaDoc

Schlecht:

/** @param x die Zahl x */
void methode(int x) { ... }

Gut:

/**
 * Berechnet die Fakultaet einer Zahl.
 * @param n Die Zahl, deren Fakultaet berechnet wird (n >= 0)
 * @return Die Fakultaet von n (n!)
 */
void methode(int n) { ... }
