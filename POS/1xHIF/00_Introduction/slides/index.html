Was ist Programmieren?

  • Ein Programm ist eine Folge von Befehlen
  • Der Computer führt diese Befehle Schritt für Schritt aus
  • Analogie: Ein Kochrezept
    • Zutaten = Daten
    • Arbeitsschritte = Befehle
    • Ergebnis = Programm-Output

Algorithmen

Ein Algorithmus ist eine eindeutige, schrittweise Anleitung zur Lösung eines Problems.

Beispiel: Tee kochen

  1. Wasser aufkochen
  2. Teebeutel in Tasse legen
  3. Heisses Wasser eingiessen
  4. 3 Minuten ziehen lassen
  5. Teebeutel entfernen

Java - was ist das?

  • Java ist eine Programmiersprache
  • Einmal schreiben, überall ausführen (Write Once, Run Anywhere)
  • Ermöglicht durch die JVM (Java Virtual Machine)
  • Quellcode (.java) wird zu Bytecode (.class) compiliert
  • Die JVM führt den Bytecode aus

Grundgerüst eines Java-Programms

public class HalloWelt {
    public static void main(String[] args) {
        // Hier stehen unsere Befehle
    }
}
  • Jedes Programm braucht eine Klasse (hier: HalloWelt)
  • Die main-Methode ist der Startpunkt

Die main-Methode

public static void main(String[] args)
  • public: für die JVM von aussen erreichbar
  • static: ohne Objekt aufrufbar (Details später)
  • void: gibt nichts zurück
  • String[] args: Kommandozeilen-Argumente (erstmal ignorieren)

Anweisungen

  • Jede Anweisung endet mit einem Semikolon (;)
  • Einrückung verbessert die Lesbarkeit
  • Java ist case-sensitive (Gross-/Kleinschreibung wichtig)
System.out.println("Hallo Welt!"); // korrekt
System.Out.Println("Hallo Welt!"); // Fehler!
system.out.println("Hallo Welt!"); // Fehler!

Was ist eine IDE?

  • IDE = Integrated Development Environment
  • Ein Texteditor mit vielen Hilfsfunktionen
  • IntelliJ ist unsere IDE für dieses Jahr

Vorteile: Syntax-Highlighting, Auto-Vervollständigung, Debugger, integrierte Compiler-Ausführung

IntelliJ installieren

  1. Download von jetbrains.com (Community Edition, kostenlos)
  2. Installation ausführen
  3. Beim ersten Start: Einstellungen übernehmen
  4. JDK auswählen (Java Development Kit)

Sollte bereits installiert sein - bei Fragen melden!

Erstes Projekt anlegen

  1. File -> New -> Project
  2. Java auswählen, JDK festlegen
  3. Projektnamen vergeben (z.B. "ErstesProgramm")
  4. Finish klicken
  5. src/ -> New -> Java Class: "HalloWelt"

Code schreiben & ausführen

  • Code in der Klasse schreiben
  • Gruner Pfeil (Run) neben der main-Methode klicken
  • Oder: Rechtsklick -> Run 'HalloWelt.main()'
  • Ausgabe erscheint im Console-Tab
public class HalloWelt {
    public static void main(String[] args) {
        System.out.println("Hello World!");
    }
}

Projekt-Struktur in IntelliJ

  • src/ - Unser Quellcode (.java-Dateien)
  • .idea/ - IntelliJ-Konfiguration (nicht anfassen)
  • out/ - Compilierte .class-Dateien (wird automatisch erzeugt)

println vs print

System.out.println("Erste Zeile");
System.out.print("Zweite ");
System.out.println("Zeile");

Ausgabe:

Erste Zeile
Zweite Zeile
  • println = print line (mit Zeilenumbruch)
  • print = ohne Zeilenumbruch

printf - Formatierte Ausgabe

String name = "Welt";
System.out.printf("Hallo, %s!", name);

int zahl = 42;
System.out.printf("Die Antwort ist %d.", zahl);

Ausgabe:

Hallo, Welt!
Die Antwort ist 42.
  • %s = Platzhalter für einen String
  • %d = Platzhalter für eine ganze Zahl

Escape-Sequenzen

Spezielle Zeichen mit Backslash (\)

System.out.println("Erste Zeile\nZweite Zeile");
System.out.println("Tab:\tEingerückt");
System.out.println("Backslash: \\");
System.out.println("Anführungszeichen: \"");

Ausgabe:

Erste Zeile
Zweite Zeile
Tab:    Eingerückt
Backslash: \
Anführungszeichen: "

Häufige Syntax-Fehler

Fehlendes Semikolon

System.out.println("Hallo") // FEHLT ;

Falsche Gross-/Kleinschreibung

System.out.println("Hallo"); // korrekt
system.out.println("Hallo"); // Fehler!

Fehlende Klammer

public static void main(String[] args // fehlt )

Compiler-Fehler lesen

HalloWelt.java:3: error: ';' expected
    System.out.println("Hallo")
                               ^
  • Dateiname.java:Zeile - Wo ist der Fehler?
  • error: ... - Was fehlt?
  • ^ - Genaü Position in der Zeile

Keine Panik! Fehler sind normal - jeder Programmierer hat sie täglich.

Debugging-Strategien

  • Console lesen - Die Fehlermeldung sagt genau, was falsch ist
  • Zeile kommentieren - Finde heraus, welche Zeile den Fehler verursacht
  • Nachfragen - Keine Scheu, den Lehrer oder Mitschüler zu fragen
  • Ausgangslage vergleichen - Was hat sich seit dem letzten Durchlauf geändert?
