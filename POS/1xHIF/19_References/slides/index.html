19 - Referenzen

POS1 - 1AHIF

Wiederholung: Kapselung

private / public

  • private Attribute: Zugriff nur innerhalb der Klasse
  • public Methoden: Zugriff von aussen

Getter und Setter

  • Getter: public int getAlter() { ... }
  • Setter: public void setAlter(int alter) { ... }

public class Auto {
    private String marke;
    private int baujahr;

    public String getMarke() { return marke; }
    public void setMarke(String marke) { this.marke = marke; }
}

Primitive vs. Referenz-Typen

Primitive Typen

  • int, double, boolean, char
  • Speichern direkt den Wert
  • Keine Methodenaufrufe moeglich

Referenz-Typen

  • String, Auto, Person, ...
  • Speichern eine Referenz (Adresse) auf das Objekt
  • Methodenaufrufe moeglich: person.getName()

int zahl = 42;           // primitiv
String text = "Hallo";   // Referenz
Auto meinAuto = new Auto("BMW"); // Referenz

Speichermodell: Stack vs. Heap

Stack (Stapelspeicher)

  • Lokale Variablen (primitiv)
  • Referenzen auf Heap-Objekte
  • Kurzlebig, Methodenaufruf-Ebene

Heap (Haldenspeicher)

  • Objekte (Instanzen von Klassen)
  • Attributwerte
  • Lebt bis zur Garbage Collection

int a = 10;          // a auf dem Stack, Wert 10
Auto b = new Auto(); // b auf Stack (Referenz),
                      // Auto-Objekt auf dem Heap

Zwei Variablen, ein Objekt


Person p1 = new Person("Anna");
Person p2 = p1;  // p2 zeigt auf DASSELBE Objekt

p2.setName("Maria");
System.out.println(p1.getName()); // "Maria"

Zwei Referenzen koennen auf dasselbe Objekt zeigen. Aenderungen uber eine Referenz sind auch uber die andere sichtbar.


// Anders bei primitiven Typen:
int x = 5;
int y = x;
y = 10;
System.out.println(x); // 5 (unabhaengig!)

null - keine Referenz

  • null bedeutet: Die Variable zeigt auf kein Objekt
  • Eine nicht initialisierte Objektvariable hat den Wert null

Person person = null;
System.out.println(person.getName());
// NullPointerException!

Vor dem Methodenaufruf prufen, ob die Referenz null ist.


if (person != null) {
    System.out.println(person.getName());
} else {
    System.out.println("Keine Person vorhanden");
}

Objekte als Attribute

Eine Klasse kann Objekte einer anderen Klasse als Attribut enthalten.


public class Person {
    private String name;
    private Adresse adresse;  // Objekt als Attribut
}


public class Adresse {
    private String strasse;
    private String plz;
    private String ort;
}

Dies nennt man auch Assoziation oder hat-ein-Beziehung.

Beispiel: Adresse als Attribut von Person


Adresse adr = new Adresse("Hauptstr. 1", "1050", "Wien");
Person person = new Person("Anna", 25, adr);

System.out.println(person.getName());
System.out.println(person.getAdresse().getOrt());

Kettenzugriff: person.getAdresse().getOrt()

Live-Coding: Person mit Adresse

Implementiere die Klassen Person und Adresse:

  • Adresse: strasse, plz, ort, Konstruktor, Getter
  • Person: name, alter, adresse, Konstruktor, Getter, toString()

Adresse a = new Adresse("Schulgasse 1", "1050", "Wien");
Person p = new Person("Max", 16, a);
System.out.println(p);
// Max (16), Schulgasse 1, 1050 Wien

Objekte als Parameter

Objekte koennen als Parameter an Methoden ubergeben werden.


public class Person {
    public void umziehen(Adresse neueAdresse) {
        this.adresse = neueAdresse;
    }
}


Adresse neueAdr = new Adresse("Berggasse 5", "1090", "Wien");
person.umziehen(neueAdr);
System.out.println(person.getAdresse());
// Berggasse 5, 1090 Wien

Es wird die Referenz auf das Objekt ubergeben.

NullPointerException vermeiden

NullPointerException tritt auf, wenn man eine Methode auf einer null-Referenz aufruft.


public void printAdresse() {
    if (this.adresse != null) {
        System.out.println(this.adresse);
    } else {
        System.out.println("Keine Adresse vorhanden");
    }
}


// Mit Ternary-Operator:
public String getAdresseOrt() {
    return adresse != null ? adresse.getOrt() : "Unbekannt";
}

Dungeon Escape: Raum-Verweise

Jeder Raum kennt seine Nachbar-Raume:


public class Raum {
    private String name;
    private String beschreibung;
    private Raum norden;
    private Raum sueden;
    private Raum osten;
    private Raum westen;

    public void setNorden(Raum raum) { this.norden = raum; }
    public Raum getNorden() { return norden; }
    // analog fuer sueden, osten, westen
}

Dungeon Escape: Spieler.aktuellerRaum


public class Spieler {
    private String name;
    private Raum aktuellerRaum;

    public Spieler(String name, Raum startRaum) {
        this.name = name;
        this.aktuellerRaum = startRaum;
    }

    public void geheNorden() {
        if (aktuellerRaum.getNorden() != null) {
            aktuellerRaum = aktuellerRaum.getNorden();
            System.out.println("Du gehst nach Norden.");
        } else {
            System.out.println("Dort ist keine Tur.");
        }
    }
}

Der Spieler bewegt sich, indem er aktuellerRaum andert.

Dungeon Escape Aufgabe

Baue ein kleines Dungeon mit 4 Raumen:

  • Startraum, Wald, Turm, Schatzraum
  • Verlinke die Raume mit setNorden(), setSueden(), etc.
  • Der Spieler startet im Startraum
  • Gib die aktuellen Ausgange aus: "Du kannst nach: norden, osten"
  • Implementiere geheNorden(), geheSueden(), geheOsten(), geheWesten()

Zusammenfassung

  • Referenz-Typen speichern Adressen von Objekten, primitive Typen speichern Werte
  • Stack: lokale Variablen (primitiv) + Referenzen; Heap: Objekte
  • null = keine Referenz - vor dem Zugriff prufen
  • Objekte koennen andere Objekte als Attribute enthalten (Assoziation)
  • Objekte werden als Referenz an Methoden ubergeben
