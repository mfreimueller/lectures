Wiederholung: OO-Grundlagen

Die vier Saulen der OO

  • Kapselung: private Attribute, public Methoden
  • Vererbung: Klasse erbt von einer anderen
  • Polymorphie: Gleiche Schnittstelle, unterschiedliche Implementierung
  • Abstraktion: Komplexitat verstecken

Was ist UML?

UML = Unified Modeling Language

  • Grafische Sprache zur Modellierung von Software
  • Wichtigstes Diagramm: Klassendiagramm
  • Zeigt Klassen, Attribute, Methoden und Beziehungen
  • Hilft beim Planen und Dokumentieren von Code

Klassendiagramm: Aufbau


+---------------------+
| Auto                |  <- Klassenname (fett)
+---------------------+
| - marke: String     |  <- Attribute
| - baujahr: int      |
+---------------------+
| + Auto(m,s,b)       |  <- Konstruktor
| + hupen(): void     |  <- Methoden
| + getMarke(): String|
+---------------------+

Sichtbarkeiten in UML

+ public

Zugriff von uberall: + getMarke(): String

- private

Zugriff nur innerhalb der Klasse: - marke: String

# protected

Zugriff in Klasse + Unterklassen: # version: int

Datentypen in UML

Nach dem Doppelpunkt kommt der Datentyp:


+---------------------+
| Person              |
+---------------------+
| - name: String      |
| - alter: int        |
| - grösse: double   |
+---------------------+
| + getAlter(): int   |
+---------------------+

Ruckgabetyp einer Methode steht ebenfalls nach Doppelpunkt.

Assoziationen

Linie = Beziehung zwischen Klassen


+--------+          +----------+
| Person |----------| Adresse  |
+--------+   hat    +----------+

Assoziation lesen

  • Person HAT EINE Adresse
  • Im Code: Person hat Attribut vom Typ Adresse

Multiplizitaten

Geben an, wie viele Objekte in Beziehung stehen:


+--------+ 1      0..* +----------+
| Kunde  |------------| Bestellung|
+--------+  hat       +----------+
  • 1 = genau eines
  • 0..* = null bis viele
  • 1..* = mindestens eines
  • 0..1 = optional (null oder eins)

Live-Coding: UML -> Code

Gegeben ist ein UML-Diagramm:


+---------------------+
| Buch                |
+---------------------+
| - titel: String     |
| - autor: String     |
+---------------------+
| + Buch(t,a)         |
| + getTitel(): String|
| + toString(): String|
+---------------------+


public class Buch {
    private String titel;
    private String autor;

    public Buch(String titel, String autor) {
        this.titel = titel;
        this.autor = autor;
    }

    public String getTitel() { return titel; }

    @Override
    public String toString() {
        return titel + " von " + autor;
    }
}

Gegner-Klasse aus UML implementieren


+---------------------+
| Gegner              |
+---------------------+
| - name: String      |
| - hp: int           |
| - schaden: int      |
+---------------------+
| + Gegner(n,h,s)     |
| + angreifen(): String|
| + getHp(): int      |
| + setHp(int): void  |
+---------------------+


public class Gegner {
    private String name;
    private int hp;
    private int schaden;

    public Gegner(String name, int hp, int schaden) {
        this.name = name;
        this.hp = hp;
        this.schaden = schaden;
    }

    public String angreifen() {
        return name + " greift an und verursacht " + schaden + " Schaden!";
    }

    public int getHp() { return hp; }
    public void setHp(int hp) { this.hp = hp; }
}

  • Wenn der Spieler einen Raum betritt, in dem ein Gegner ist, greift der Gegner an
  • Der Spieler verliert HP (Lebenspunkte)

Zusammenfassung

  • UML-Klassendiagramme visualisieren die Struktur von OO-Programmen
  • Klasse: Kasten mit Name, Attribute, Methoden
  • Sichtbarkeiten: + (public), - (private)
  • Assoziationen: Linien zwischen Klassen (mit Multiplizitaten)
  • UML kann in Code ubersetzt werden, und Code kann in UML dargestellt werden
