## Learning Objectives

- Configure Spring Boot applications using `application.yml` and `application.properties`
- Create typed configuration classes with `@ConfigurationProperties`
- Use `@Profile` to define environment-specific beans and settings
- Activate profiles via command line, environment variables, or config files
- Externalize configuration for different environments (dev, test, prod)

## Presentation (slides/)

Covers Spring Boot configuration mechanisms: properties files, YAML, `@Value`, `@ConfigurationProperties`, and profiles for environment-specific configuration.

## Katas (katas/)

1. **K1: Configuration Properties** — Create a typed config class for application settings.
2. **K2: Profiles** — Define dev, test, and prod profiles with different datasources.
