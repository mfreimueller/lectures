## Learning Objectives

- Distinguish between association, aggregation, and composition in UML
- Model inheritance and realization relationships in class diagrams
- Apply the core DDD concepts: Ubiquitous Language and Bounded Context
- Differentiate between Entities and Value Objects
- Brainstorm and evaluate potential domains for the group project

## Presentation (slides/)

Recaps UML class diagram basics, then focuses on the three relationship types (association, aggregation, composition) with Java examples. Introduces Domain-Driven Design concepts including Ubiquitous Language and Bounded Context. Connects to the project by guiding domain brainstorming.

## Katas (katas/)

1. **K1: UML from Code** — Given Java classes, draw the UML class diagram with correct relationships.
2. **K2: DDD Event Storming** — Simplified event storming for a library domain using sticky notes.
3. **K3: Domain Brainstorming** — Brainstorm 3 potential domains for the group project and evaluate them against requirements.
