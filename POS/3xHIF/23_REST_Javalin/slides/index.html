Review: REST Recap

  • REST uses HTTP methods for CRUD on resources
  • Resources identified by URIs (e.g., /api/users)
  • Standard status codes: 200 OK, 201 Created, 404 Not Found
  • Java HttpClient can send requests and receive responses

Review: From Client to Server

  • Last week: we built HTTP clients that consume APIs
  • This week: we build the server side
  • Same protocol, same REST principles, different perspective

Client (HttpClient) ← HTTP → Server (Javalin)

What is Javalin?

  • Lightweight Java web framework
  • Built on top of Jetty (embedded server)
  • Simple, expressive routing API
  • Built-in support for JSON, WebSockets, OpenAPI

Setting Up Javalin

Add dependency to pom.xml:


implementation 'io.javalin:javalin:6.1.3'
implementation 'com.fasterxml.jackson.core:jackson-databind:2.16.0'

Minimal server:


var app = Javalin.create(ctx -> {
    ctx.jsonMapper(new JacksonMapper());
}).start(7070);

app.get("/", ctx -> ctx.result("Hello World"));

Routing Basics


app.get("/hello", ctx -> {
    ctx.result("Hello, GET!");
});

app.post("/hello", ctx -> {
    ctx.result("Hello, POST!");
});

app.put("/hello", ctx -> {
    ctx.result("Hello, PUT!");
});

app.delete("/hello", ctx -> {
    ctx.result("Hello, DELETE!");
});

Route Handlers

  • Handler is a lambda: ctx -> { ... }
  • Context object provides request/response access
  • ctx.result() - set response body (String)
  • ctx.json() - serialize object to JSON
  • ctx.status() - set HTTP status code

app.get("/api/status", ctx -> {
    ctx.json(Map.of("status", "running", "version", 1.0));
});

JSON Serialization

Javalin uses Jackson for JSON by default:


public record Todo(int id, String title, boolean completed) {}

app.get("/api/todos", ctx -> {
    var todos = List.of(
        new Todo(1, "Learn Javalin", false),
        new Todo(2, "Build REST API", true)
    );
    ctx.json(todos);
});

Response: [{"id":1,"title":"Learn Javalin","completed":false}, ...]

Receiving JSON


app.post("/api/todos", ctx -> {
    var todo = ctx.bodyAsClass(Todo.class);
    // todo is deserialized from request body
    System.out.println("Received: " + todo);
    ctx.status(201).json(todo);
});

// Usage: curl -X POST -H "Content-Type: application/json" \
//   -d '{"id":1,"title":"test","completed":false}' \
//   http://localhost:7070/api/todos

Path Parameters


app.get("/api/todos/{id}", ctx -> {
    var id = ctx.pathParamAsClass("id", Integer.class).get();
    var todo = todoService.findById(id);
    if (todo != null) {
        ctx.json(todo);
    } else {
        ctx.status(404).result("Todo not found");
    }
});

// GET /api/todos/42
// ctx.pathParam("id") -> "42"

Query Parameters


app.get("/api/todos", ctx -> {
    var completed = ctx.queryParam("completed");

    if (completed != null) {
        var filter = Boolean.parseBoolean(completed);
        var filtered = todos.stream()
            .filter(t -> t.completed() == filter)
            .toList();
        ctx.json(filtered);
    } else {
        ctx.json(todos);
    }
});

// GET /api/todos?completed=true

CRUD: Create


app.post("/api/todos", ctx -> {
    var todo = ctx.bodyAsClass(Todo.class);
    var created = todoService.create(todo);
    ctx.status(201).json(created);
});

CRUD: Read and Update


// Read all
app.get("/api/todos", ctx -> ctx.json(todoService.findAll()));

// Read one
app.get("/api/todos/{id}", ctx -> {
    var todo = todoService.findById(ctx.pathParamAsClass("id", Integer.class).get());
    if (todo != null) ctx.json(todo);
    else ctx.status(404);
});

// Update
app.put("/api/todos/{id}", ctx -> {
    var todo = ctx.bodyAsClass(Todo.class);
    var updated = todoService.update(ctx.pathParamAsClass("id", Integer.class).get(), todo);
    if (updated != null) ctx.json(updated);
    else ctx.status(404);
});

CRUD: Delete


app.delete("/api/todos/{id}", ctx -> {
    var id = ctx.pathParamAsClass("id", Integer.class).get();
    var deleted = todoService.delete(id);
    if (deleted) ctx.status(204);
    else ctx.status(404);
});

// Testing with curl:
// curl -X DELETE http://localhost:7070/api/todos/1

Summary

  • Javalin provides a simple API for building REST servers
  • Routes map HTTP methods + paths to handler lambdas
  • Jackson handles JSON serialization/deserialization
  • Path params and query params enable dynamic routing
  • CRUD endpoints follow REST conventions

Best Practices

  • Use plural nouns for resource paths (/todos, not /todo)
  • Return appropriate status codes (201 for create, 204 for delete)
  • Validate input before processing
  • Handle errors gracefully with try-catch
  • Test endpoints with curl or a REST client
