Review: DTOs and Validation

  • DTOs separate API contracts from domain models
  • Custom annotations + reflection enable reusable validation
  • Exception mappers centralize error handling
  • Structured error responses help client developers

Review: Current Architecture

  • All code lives in route handlers (big lambda functions)
  • Hard to test, hard to maintain, hard to extend
  • Business logic mixed with HTTP concerns

// Monolithic handler - everything in one place
app.get("/api/users/{id}", ctx -> {
    // validation, business logic, data access all here
    var id = Long.parseLong(ctx.pathParam("id"));
    // ... query database, format response, handle errors
});

The Problem with Monoliths

  • Violates Single Responsibility Principle
  • Code duplication across handlers
  • Cannot test business logic without HTTP
  • Changing data source affects everything

Solution: Separate concerns into layers

Three-Layer Architecture


Client (HTTP)
    |
    v
+------------------+
|  Controller      |  HTTP routing, request/response handling
+------------------+
    |
    v
+------------------+
|  Service         |  Business logic, validation, orchestration
+------------------+
    |
    v
+------------------+
|  Repository      |  Data access (in-memory, JDBC, JPA)
+------------------+
    |
    v
   Database

Layer Responsibilities

LayerResponsibilityDepends On
ControllerHTTP handling, routing, JSON serializationService (interface)
ServiceBusiness logic, validation, transactionsRepository (interface)
RepositoryData access, CRUD operationsDomain entities

Repository Layer


// Interface (abstraction)
public interface UserRepository {
    Optional<User> findById(Long id);
    List<User> findAll();
    User save(User user);
    boolean delete(Long id);
}

// In-memory implementation
public class InMemoryUserRepository implements UserRepository {
    private final Map<Long, User> users = new ConcurrentHashMap<>();
    private long nextId = 1;

    public User save(User user) {
        var saved = new User(nextId++, user.username(), user.email());
        users.put(saved.id(), saved);
        return saved;
    }
    // ... other methods
}

Service Layer


// Interface
public interface UserService {
    UserResponse createUser(CreateUserRequest request);
    UserResponse getUser(Long id);
    List<UserResponse> getAllUsers();
    void deleteUser(Long id);
}

// Implementation depends on Repository interface
public class UserServiceImpl implements UserService {
    private final UserRepository repository;

    public UserServiceImpl(UserRepository repository) {
        this.repository = repository; // Constructor injection
    }

    public UserResponse createUser(CreateUserRequest request) {
        // Business logic: validation, mapping, rules
        var user = repository.save(new User(request.username(), request.email()));
        return new UserResponse(user.id(), user.username(), user.email());
    }
    // ... other methods
}

Controller Layer


// Controller wires everything together
public class UserController {
    private final UserService service;

    public UserController(UserService service) {
        this.service = service;
    }

    public void registerRoutes(Javalin app) {
        app.get("/api/users", ctx -> {
            ctx.json(service.getAllUsers());
        });

        app.get("/api/users/{id}", ctx -> {
            var id = Long.parseLong(ctx.pathParam("id"));
            var user = service.getUser(id);
            if (user == null) ctx.status(404);
            else ctx.json(user);
        });

        app.post("/api/users", ctx -> {
            var request = ctx.bodyAsClass(CreateUserRequest.class);
            var created = service.createUser(request);
            ctx.status(201).json(created);
        });
    }
}

Constructor Injection

  • Dependencies are passed through the constructor
  • No framework needed -- pure Java!
  • Makes dependencies explicit and testable

// Wiring everything together
var repository = new InMemoryUserRepository();
var service = new UserServiceImpl(repository);
var controller = new UserController(service);

var app = Javalin.create().start(7070);
controller.registerRoutes(app);

Testing with Layers


// Service test with mock repository
class UserServiceTest {
    @Test
    void testCreateUser() {
        var repo = new InMemoryUserRepository();
        var service = new UserServiceImpl(repo);

        var request = new CreateUserRequest("alice", "alice@test.com");
        var result = service.createUser(request);

        assertThat(result.username()).isEqualTo("alice");
        assertThat(result.id()).isNotNull();
    }
}

No HTTP server needed to test business logic!

Summary

  • Three-layer architecture separates HTTP, business logic, and data access
  • Each layer depends on interfaces, not implementations
  • Constructor injection makes dependencies explicit
  • Layers enable testing business logic without HTTP
  • Swapping implementations (e.g., in-memory to JDBC) is transparent
