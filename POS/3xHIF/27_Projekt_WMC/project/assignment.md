# POS-WMC Project - Week 27: Kickoff, Setup, Architecture Planning

## Task

Choose a personal domain for your POS-WMC project, define the database model, and set up the project skeleton.

### Requirements

1. Choose a domain that interests you (library, gym, restaurant, hotel, e-commerce, etc.)
2. Define at least 6 database tables with appropriate columns and data types
3. Include at least one 1:1 relationship
4. Include at least one 1:n relationship
5. Include at least one inheritance case (using JPA inheritance strategies)
6. Set up a Maven project with Javalin, Hibernate/JPA, PostgreSQL, and Jackson dependencies
7. Create the layered architecture package structure (controller, service, repository, model, config)
8. Write a plan.md describing your domain, model, and architecture

### Example Output

A project directory with:
- `plan.md` describing the domain and architecture
- `pom.xml` with all dependencies
- Source package structure with empty placeholder classes

### Extension

Add a Docker Compose file for PostgreSQL and a basic Hibernate configuration with `persistence.xml`.
