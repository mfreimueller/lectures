# Speaker Notes — 33: Projekt WMC Final

**Woche 37** | **Typ: Projekt — Sprint 6**

---

## 0. Check-in (15 min)

- Stand-up: Was ist noch offen?
- Letzte Probleme besprechen
- Präsentationen vorbereiten

## 1. Fokus dieser Woche (10 min)

- **Finalisierung & Präsentation**
- Code aufräumen
- README schreiben
- Präsentation vorbereiten

## 2. Checklist (20 min)

- [ ] Alle CRUD-Endpoints funktionieren
- [ ] Authentifizierung funktioniert
- [ ] Tests geschrieben und grün
- [ ] README mit Setup-Anleitung
- [ ] ER-Diagramm im README
- [ ] API-Dokumentation (Endpoints, Request/Response)
- [ ] Code-Qualität (SOLID, Naming, keine Duplikation)
- [ ] Git-History sauber (aussagekräftige Commits)

## 3. Präsentationen (2h)

- **Pro Projekt:** 10 min Präsentation + 5 min Q&A
- **Inhalt:**
  - Domain vorstellen
  - Architektur zeigen
  - Live-Demo der API
  - Lessons Learned
  - Challenges & Solutions

## 4. Feedback & Bewertung (15 min)

- Bewertungskriterien transparent machen
- Konstruktives Feedback geben
- Stärken hervorheben

## 5. Wrap-up (10 min)

- Nächste Woche: Retrospective & Outlook
- Feierabend!

---

## Bewertungskriterien

- **Funktionalität** (40%) — Alle Anforderungen erfüllt?
- **Architektur** (20%) — Layered Architecture, SOLID?
- **Testing** (15%) — Tests geschrieben, sinnvoll?
- **Code-Qualität** (15%) — Lesbarkeit, Naming, Struktur?
- **Präsentation** (10%) — Klar, verständlich, Demo?
