NT-Architektur

  • Windows NT (New Technology) ist die gemeinsame Basis seit Windows NT 3.1 (1993)
  • Hybrid-Kernel: Kombination aus Mikrokernel- und monolithischen Elementen
  • Wichtige Schichten: Hardware Abstraction Layer (HAL), Kernel, Executive, Subsysteme
  • Eigenschaften: präemptives Multitasking, Speicherschutz, Multiprozessor-Unterstützung

Startmenü und Taskleiste

  • Startmenü: Zentrale Anlaufstelle zum Starten von Programmen, öffnen von Ordnern und Systemeinstellungen
  • Taskleiste: Zeigt laufende Programme an; Fixieren von Programmen möglich
  • Infobereich (System Tray): Uhrzeit, Lautstärke, Netzwerk, Benachrichtigungen
  • Aufgabenansicht (Task View): Virtuelle Desktops und Fensterwechsel (Win + Tab)
  • Suchfunktion: Suche nach Apps, Dokumenten und Web (Win + S)

Fensterverwaltung

  • Aero Snap: Fenster an Bildschirmseiten oder -ecken ziehen für automatische Größenanpassung (Win + Pfeiltasten)
  • Minimieren, Maximieren, Schließen: Standard-Fenstersteuerung oben rechts
  • Taskleisten-Gruppierung: Mehrere Fenster derselben App werden gruppiert
  • Alt + Tab: Schnelles Wechseln zwischen geöffneten Fenstern
  • Rechtsklick auf Taskleiste → Taskleisteneinstellungen für Anpassungen

NTFS-Dateisystem

New Technology File System

Master File Table (MFT)

  • Jede Datei und jedes Verzeichnis wird durch einen MFT-Eintrag repräsentiert
  • Jeder Eintrag ist typischerweise 1 KiB groß
  • Kleine Dateien (< ~600 Byte) werden direkt im MFT-Eintrag gespeichert (residente Daten)
  • Größere Dateien enthalten Verweise auf Extents (zusammenhängende Speicherbereiche)
  • Die MFT ist die zentrale Verwaltungsstruktur von NTFS

NTFS-Dateiberechtigungen

  • Jede Datei/Ordner hat eine Access Control List (ACL)
  • Jeder ACL-Eintrag (ACE) besteht aus: Benutzer/Gruppe + Berechtigung (Allow/Deny)
  • Wichtige Berechtigungen: Vollzugriff, Ändern, Lesen & Ausführen, Ordnerinhalt auflisten, Lesen, Schreiben
  • Vererbung: Untergeordnete Objekte erben standardmäßig die Berechtigungen des übergeordneten Ordners
  • Effektive Berechtigung: Summe aller Allow- minus Deny-Einträge (Deny hat Vorrang)

Wichtige Bereiche: System, Geräte, Netzwerk, Konten, Update & Sicherheit

Datenträgerverwaltung und Geräte-Manager

  • Datenträgerverwaltung (diskmgmt.msc):
    • Anzeigen und Verwalten von Festplatten, Partitionen und Volumes
    • Partitionen erstellen, formatieren, erweitern, verkleinern
    • Laufwerksbuchstaben zuweisen, Volume-Erstellung (einfach, spanned, striped, mirrored)
  • Geräte-Manager (devmgmt.msc):
    • Übersicht aller installierten Hardware-Komponenten
    • Treiber aktualisieren, deinstallieren, deaktivieren
    • Konflikte und Fehler erkennen (gelbes Ausrufezeichen)

Ereignisanzeige (Event Viewer)

  • Aufruf: eventvwr.msc oder Win + R → eventvwr
  • Protokolliert System-, Sicherheits- und Anwendungsereignisse
  • Ereignistypen: Information, Warnung, Fehler, Erfolgs- und Fehlerüberwachung
  • Windows-Protokolle: Anwendung, Sicherheit, Setup, System, Weitergeleitete Ereignisse
  • Unverzichtbar zur Fehleranalyse nach Systemabstürzen oder Treiberproblemen

Lokale Benutzerverwaltung

  • Lokale Benutzer und Gruppen (lusrmgr.msc): Anlegen, Bearbeiten und Löschen von lokalen Benutzerkonten
  • Standardbenutzer: Administrator (vollständige Kontrolle) vs. Standardbenutzer (eingeschränkt)
  • Gruppen: Z. B. Administratoren, Benutzer, Gäste, Remotedesktopbenutzer
  • User Account Control (UAC): Fragt bei Administrator-Aktionen nach Bestätigung – schützt vor unautorisierter Software
  • Passwortrichtlinien: Mindestlänge, Komplexität, Gültigkeitsdauer (über Gruppenrichtlinien konfigurierbar)

Task-Manager (taskmgr.exe)

  • Aufruf: Strg + Shift + Esc oder Rechtsklick auf Taskleiste
  • Standard-Ansicht: Nur laufende Anwendungen
  • Detail-Ansicht („Mehr Details") mit mehreren Registerkarten:
    • Prozesse: Übersicht aller laufenden Prozesse mit CPU-, RAM-, Datenträger- und Netzwerkauslastung
    • Leistung: Grafische Darstellung der Auslastung von CPU, RAM, Datenträgern, Netzwerk
    • App-Verlauf: Ressourcennutzung von Store-Apps
    • Autostart: Programme, die beim Systemstart automatisch ausgeführt werden
    • Benutzer: Anwendungen und Dienste pro angemeldetem Benutzer
    • Details: Erweiterte Prozessinformationen (PID, Priorität, CPU-Zeit)
    • Dienste: Verwaltung von Windows-Diensten (Start/Stopp/Neustart)
