Benutzerverwaltung (1) – useradd, userdel

useradd -m -s /bin/bash max     # Neuen Benutzer anlegen
# -m = Home-Verzeichnis erstellen, -s = Shell festlegen

passwd max                      # Passwort setzen

userdel -r max                  # Benutzer + Home-Verzeichnis löschen

Benutzerdaten werden in /etc/passwd gespeichert:

max:x:1001:1001:Max Mustermann:/home/max:/bin/bash

Benutzerverwaltung (2) – Gruppen

groupadd schueler           # Neue Gruppe anlegen
groupadd lehrer

usermod -aG schueler max    # Benutzer zu Gruppe hinzufügen
usermod -aG lehrer max

groups max                  # Gruppen eines Benutzers anzeigen

groupdel schueler           # Gruppe löschen

Gruppendaten in /etc/group

Benutzerverwaltung (3) – sudo und sudoers

  • sudo = superuser do – Befehle als root ausführen
  • Nur Benutzer in /etc/sudoers dürfen sudo verwenden
  • Bearbeiten mit visudo (niemals direkt editieren!)
sudo apt update              # Befehl als root
sudo -i                      # permanente root-Shell
visudo                       # sudoers-Datei bearbeiten

Typische sudoers-Zeile: max ALL=(ALL:ALL) ALL

Paketmanager (apt) – update und upgrade

apt (Advanced Package Tool) – der Paketmanager von Debian/Ubuntu

sudo apt update              # Paketliste aktualisieren
sudo apt upgrade             # Alle Pakete aktualisieren
sudo apt full-upgrade        # Mit Abhängigkeitsauflösung
  • apt update lädt die aktuellen Paketlisten aus den Repositories
  • apt upgrade installiert neuere Versionen aller Pakete
  • Repositories in /etc/apt/sources.list

Paketmanager (apt) – install, remove, dpkg

sudo apt install htop        # Paket installieren
sudo apt remove htop         # Paket deinstallieren
sudo apt purge htop          # Inklusive Konfigurationsdateien

sudo apt search "text"       # Nach Paketen suchen
sudo apt show htop           # Paketdetails anzeigen

sudo dpkg -i paket.deb       # .deb-Paket manuell installieren
sudo dpkg -l                 # Alle installierten Pakete auflisten

Prozessverwaltung (1) – ps, top

ps aux                       # Alle laufenden Prozesse anzeigen
ps aux | grep firefox        # Bestimmten Prozess suchen

top                          # Echtzeit-Ansicht der Prozesse
htop                         # Verbesserte Version von top
  • PID = Process ID (eindeutige Nummer)
  • PPID = Parent Process ID
  • top zeigt CPU-, RAM-Nutzung und laufende Prozesse
  • Beenden von top mit der Taste q

Prozessverwaltung (2) – kill, nice, fg/bg

kill 1234                    # Prozess mit PID 1234 beenden
kill -9 1234                 # Prozess erzwingen beenden (SIGKILL)
kill -15 1234                # Prozess höflich beenden (SIGTERM)

nice -n 10 ./skript.sh       # Prozess mit niedriger Priorität starten
renice 5 -p 1234             # Priorität eines laufenden Prozesses ändern

sleep 100 &                   # Prozess im Hintergrund starten
fg %1                        # Hintergrundprozess in Vordergrund holen
bg %1                        # Prozess in Hintergrund schicken

systemd (1) – systemctl, Unit-Dateien

systemd ist das Init-System moderner Linux-Distributionen

systemctl status ssh         # Status eines Dienstes anzeigen
systemctl start ssh          # Dienst starten
systemctl stop ssh           # Dienst stoppen
systemctl restart ssh        # Dienst neustarten
systemctl enable ssh         # Dienst automatisch starten (Boot)
systemctl disable ssh        # Automatischen Start deaktivieren

Unit-Dateien liegen in /etc/systemd/system/

systemd (2) – journalctl

journalctl zeigt die System-Logs von systemd an

journalctl                   # Alle Logs anzeigen
journalctl -u ssh            # Logs eines bestimmten Dienstes
journalctl -f                # Logs in Echtzeit verfolgen
journalctl --since "1 hour ago"  # Logs der letzten Stunde
journalctl -p err            # Nur Fehlermeldungen
  • Logs werden binär in /var/log/journal/ gespeichert
  • Mit dmesg können Kernel-Ringpuffer-Nachrichten angezeigt werden

Netzwerkkonfiguration (1) – ip addr, ping

ip addr show                 # IP-Adressen und Netzwerkschnittstellen
ip link set eth0 up/down     # Schnittstelle aktivieren/deaktivieren

ping 8.8.8.8                 # Erreichbarkeit eines Hosts prüfen
ping -c 4 google.com        # Nur 4 Pakete senden

hostname                     # Rechnername anzeigen
hostnamectl set-hostname neuer-name  # Rechnername ändern

Neuere Netzwerkbefehle ersetzen das veraltete ifconfig

Netzwerkkonfiguration (2) – netstat, DHCP

ss -tuln                     # Offene Ports anzeigen (modern)
netstat -tuln                # Offene Ports anzeigen (klassisch)

ip route show                # Routing-Tabelle anzeigen

# DHCP: Automatische IP-Vergabe
# Konfiguration in /etc/netplan/ (Ubuntu)
# oder /etc/network/interfaces (Debian)
  • Standardmäßig bezieht Linux die IP per DHCP
  • DHCP = Dynamic Host Configuration Protocol
  • Statische IPs werden in Netplan- oder interfaces-Dateien konfiguriert

Log-Dateien

Linux speichert Systemmeldungen und Logs in /var/log/

ls /var/log                  # Alle Log-Dateien anzeigen
cat /var/log/syslog          # System-Logs
tail -f /var/log/syslog      # Logs in Echtzeit verfolgen

dmesg                        # Kernel-Ringpuffer (Hardware-Erkennung)
journalctl -xe               # systemd-Logs mit Erklärungen
Log-DateiInhalt
/var/log/syslogAllgemeine Systemmeldungen
/var/log/auth.logAuthentifizierungsversuche
/var/log/dpkg.logPaketinstallationen

Sicherheit – sudo, ufw, SSH

# Firewall mit ufw (Uncomplicated Firewall)
sudo ufw enable              # Firewall aktivieren
sudo ufw allow 22/tcp        # SSH-Port freigeben
sudo ufw allow 80/tcp        # HTTP-Port freigeben
sudo ufw status              # Status anzeigen

# SSH-Server
sudo apt install openssh-server
sudo systemctl status ssh
sudo nano /etc/ssh/sshd_config   # SSH-Konfiguration
  • sudo beschränkt root-Zugriff auf berechtigte Benutzer
  • ufw vereinfacht die iptables-Firewall
  • SSH (Secure Shell) ermöglicht verschlüsselte Remote-Zugriffe

  • Netzwerk: ip addr, ping, ss, DHCP
  • Logs: /var/log, dmesg, journalctl
  • Sicherheit: sudo, ufw, SSH
