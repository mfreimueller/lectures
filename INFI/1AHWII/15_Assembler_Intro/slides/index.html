15 - Assembler Einführung

INFI - 1AHWII

Maschinensprache – Was ist das?

  • Die CPU versteht nur Binärcode (0 und 1)
  • Jeder Befehl ist eine Bitfolge, z. B. 10110000 01100001
  • Diese Bitfolgen heißen Maschinenbefehle
  • Maschinensprache ist prozessorabhängig (x86, ARM, RISC-V)
  • Früher wurden Programme direkt in Maschinensprache geschrieben

Assembler – Die Lösung

  • Assembler ist eine textuelle Darstellung der Maschinensprache
  • Jeder Maschinenbefehl bekommt einen Mnemonik (z. B. MOV, ADD)
  • Ein Assembler (übersetzt) den Text in Binärcode
MaschinenspracheAssembler
10110000 01100001MOV AL, 61h
00000101 00000001ADD AX, 1
  • Assembler ist lesbarer als Maschinensprache, aber immer noch hardwarenah

Die CPU-Register – Übersicht

  • Register sind schnelle Speicherplätze direkt in der CPU
  • Sie sind sehr klein (8, 16, 32 oder 64 Bit)
  • Die x86-Architektur hat allgemeine und spezielle Register
RegisterGröße (16 Bit)Verwendung
AX16 BitAllgemein (Akkumulator)
BX16 BitAllgemein (Basis)
CX16 BitAllgemein (Zähler)
DX16 BitAllgemein (Daten)

Allgemeine Register (AX, BX, CX, DX)

  • AX (Accumulator) – Arithmetik, E/A-Operationen
  • BX (Base) – Basisadresse für Speicherzugriffe
  • CX (Counter) – Schleifenzähler (LOOP, Shift)
  • DX (Data) – Datenregister, Port-Adressen

Jedes 16-Bit-Register ist in zwei 8-Bit-Teile zerlegbar:

HighLow
AHAL
BHBL
CHCL
DHDL

Spezielle Register (IP, SP, Flags)

  • IP (Instruction Pointer) – Zeigt auf den nächsten Befehl
  • SP (Stack Pointer) – Zeigt auf das oberste Stack-Element
  • Flags (Statusregister) – Einzelne Bits mit Status-Informationen

Wichtige Flags:

FlagBedeutung
ZF (Zero Flag)Ergebnis ist 0
CF (Carry Flag)Überlauf bei unsigned
SF (Sign Flag)Ergebnis ist negativ
OF (Overflow Flag)Überlauf bei signed

Der Befehlssatz – MOV

  • MOV kopiert Daten von einer Quelle zu einem Ziel
MOV AX, 42     ; Konstante 42 in AX laden
MOV BX, AX     ; AX nach BX kopieren
MOV CX, [100]  ; Wert von Speicherstelle 100 nach CX
  • Syntax: MOV Ziel, Quelle
  • Quelle und Ziel müssen gleiche Größe haben
  • Es kann nicht von Speicher zu Speicher kopiert werden

Arithmetische Befehle – ADD, SUB

  • ADD addiert zwei Werte und speichert das Ergebnis im Ziel
  • SUB subtrahiert zwei Werte
MOV AX, 10     ; AX = 10
ADD AX, 5      ; AX = 15
MOV BX, 20     ; BX = 20
SUB BX, AX     ; BX = 20 - 15 = 5
  • Die Flags (ZF, CF, SF, OF) werden automatisch aktualisiert
  • Beispiel: ADD AX, 5 ⇒ ZF = 1 wenn AX + 5 = 0

Sprungbefehle – JMP, CMP

  • JMP – unbedingter Sprung zu einer Marke (Label)
  • CMP – vergleicht zwei Werte (setzt Flags)
  • JE, JNE, JG, JL – bedingte Sprünge
CMP AX, BX     ; Vergleiche AX mit BX
JE  gleich     ; Springe zu 'gleich' wenn AX == BX
JMP weiter     ; Sonst springe zu 'weiter'

gleich:
    MOV CX, 1
weiter:
    MOV CX, 0

Aufbau eines Assembler-Programms

  • Ein Assembler-Programm ist in Segmente unterteilt
ORG 100h       ; Startadresse festlegen

.DATA          ; Datensegment
    zahl DB 10 ; Variable 'zahl' mit Wert 10

.CODE          ; Codesegment
    MOV AX, zahl
    ADD AX, 5
    RET        ; Rückkehr zum Betriebssystem
  • .DATA – deklariert Variablen und Konstanten
  • .CODE – enthält die ausführbaren Befehle

Direktiven im Assembler

  • Direktiven sind Anweisungen an den Assembler, nicht an die CPU
ORG 100h       ; Startadresse (Origin)
DB  10         ; Define Byte (8 Bit) – reserviert 1 Byte
DW  1000       ; Define Word (16 Bit) – reserviert 2 Bytes
DD  50000      ; Define Double (32 Bit)
EQU 100        ; Konstante definieren (ersetzen)
  • ORG – legt die Speicherposition fest
  • DB/DW/DD – deklarieren Daten mit bestimmter Größe
  • EQU – definiert eine Konstante (wird vom Assembler ersetzt)

Label und Sprungmarken

  • Ein Label ist ein Name für eine Speicheradresse
  • Label werden mit Doppelpunkt am Ende geschrieben
start:         ; Label 'start'
    MOV AX, 0
    MOV CX, 10

schleife:      ; Label 'schleife'
    ADD AX, CX
    LOOP schleife  ; CX--; springe wenn CX != 0

ende:          ; Label 'ende'
    RET
  • Label machen das Programm strukturierter und lesbarer
  • Sie werden für Sprünge und Schleifen verwendet

Das erste Assembler-Programm (Teil 1)

ORG 100h

.DATA
    msg DB 'Hallo Welt$'
    ergebnis DW 0

.CODE
start:
    MOV AH, 09h
    LEA DX, msg
    INT 21h
  • Das Programm gibt „Hallo Welt“ auf dem Bildschirm aus
  • INT 21h ist ein DOS-Interrupt für Bildschirmausgabe
  • LEA lädt die Adresse einer Variable

Das erste Assembler-Programm (Teil 2)

    MOV AX, 100
    ADD AX, 200
    MOV [ergebnis], AX

    MOV AH, 4Ch
    INT 21h       ; Programm beenden
  • INT 21h mit AH = 4Ch beendet das Programm
  • Das Ergebnis der Addition wird in ergebnis gespeichert

Gesamtstruktur:

  • ORG – Startadresse
  • .DATA – Variablen
  • .CODE – Befehle mit Label start:
  • INT 21h – DOS-Interrupts für Ein-/Ausgabe

Debugging und Zusammenfassung

  • Assembler-Programme können mit Debuggern (z. B. TD, GDB) analysiert werden
  • Wichtige Debugging-Techniken:
    • Schrittweise Ausführung (Step Into / Step Over)
    • Register und Speicher inspizieren
    • Breakpoints setzen
  • Häufige Fehler: Falsche Operandengröße, vergessene RET, falsche Segment-Adressen

Zusammenfassung:

  • Assembler = lesbare Version der Maschinensprache
  • Register sind schnelle CPU-interne Speicher
  • MOV, ADD, SUB, JMP, CMP sind grundlegende Befehle
  • Ein Programm besteht aus Segmenten, Direktiven und Labels
