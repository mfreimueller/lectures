Was ist ein Betriebssystem?

Definition und grundlegende Aufgaben

Definition

  • Ein Betriebssystem (OS) ist die zentrale Software eines Computers
  • Es agiert als Vermittler zwischen Hardware und Anwendungssoftware
  • Stellt eine Abstraktion der Hardware bereit
  • Beispiele: Windows, Linux, macOS, Android, iOS
„Ein Betriebssystem ist eine Sammlung von Systemprogrammen, die den Betrieb eines Rechners steuert und überwacht."

Die vier Hauptaufgaben eines Betriebssystems

  • Prozessverwaltung: Erzeugen, Planen und Beenden von Prozessen; Synchronisation und Kommunikation
  • Speicherverwaltung: Verteilung des Hauptspeichers an laufende Prozesse; virtuelle Adressräume
  • Dateisystemverwaltung: Organisation von Dateien und Verzeichnissen auf persistenten Speichermedien
  • Geräteverwaltung: Ansteuerung und Abstraktion von Ein-/Ausgabegeräten über Treiber

Kernel-Modus vs. User-Modus

Prozessoren unterstützen mindestens zwei Ausführungsmodi:

Kernel-Modus (Ring 0)User-Modus (Ring 3)
Vollständiger Hardware-ZugriffEingeschränkter Zugriff
Kann alle CPU-Instruktionen ausführenKritische Instruktionen gesperrt
Betriebssystem-Kernel läuft hierAnwendungsprogramme laufen hier
Direkter Speicherzugriff möglichNur eigener virtueller Adressraum

Systemaufrufe (Syscalls)

Systemaufrufe sind die Schnittstelle zwischen User-Space und Kernel:

  1. Anwendung ruft eine Funktion der Standardbibliothek auf (z. B. read())
  2. Die Bibliothek führt einen Trap (Software-Interrupt) aus
  3. Der Prozessor wechselt in den Kernel-Modus
  4. Der Kernel führt die gewünschte Operation aus
  5. Rückkehr in den User-Modus mit dem Ergebnis

Beispiel-Syscalls: open, read, write, fork, execve

Was ist ein Prozess?

  • Ein Prozess ist ein Programm in Ausführung
  • Enthält: Code (Text), Daten, Heap, Stack, Registerinhalte
  • Jeder Prozess hat einen eigenen virtuellen Adressraum
  • Prozesse sind voneinander isoliert – ein Absturz betrifft nur den eigenen Prozess

Abgrenzung: Ein Programm ist eine passive Datei auf der Festplatte, ein Prozess ist die aktive Ausführung dieses Programms im Speicher.

Prozesszustände (Process States)

Ein Prozess durchläuft während seiner Lebensdauer verschiedene Zustände:

ZustandBedeutung
RunningDer Prozess wird gerade von der CPU ausgeführt
ReadyDer Prozess ist bereit zur Ausführung, wartet aber auf die CPU
BlockedDer Prozess wartet auf ein Ereignis (z. B. Tastatureingabe, Festplatte)

Zustandsübergänge: Ready → Running (Scheduling), Running → Ready (Unterbrechung), Running → Blocked (E/A), Blocked → Ready (Ereignis tritt ein).

Process Control Block (PCB)

Für jeden Prozess führt das Betriebssystem einen PCB (auch Prozessdeskriptor):

  • Prozess-ID (PID)
  • Prozesszustand (Running, Ready, Blocked)
  • Program Counter (nächste auszuführende Adresse)
  • CPU-Register (müssen bei Unterbrechung gesichert werden)
  • Speicherverwaltungs-Informationen (Seitentabellen etc.)
  • geöffnete Dateien, Signale, Priorität

Der PCB wird im Kernel-Space gespeichert und ist für User-Prozesse nicht sichtbar.

Virtueller Speicher

  • Jeder Prozess sieht einen eigenen, zusammenhängenden Adressraum
  • Dieser muss nicht physisch zusammenhängen
  • Vorteile: Isolation (Prozesse stören einander nicht), Vereinfachung (Programme müssen nicht um Speicherplatz konkurrieren)
  • Die Adressen, die ein Programm verwendet, sind virtuelle Adressen
  • Die MMU (Memory Management Unit) übersetzt diese in physische Adressen

Paging und Seitentabelle

  • Der virtuelle Speicher ist in gleich große Blöcke (Seiten / Pages) eingeteilt
  • Der physische Speicher ist in gleich große Rahmen (Frames) eingeteilt (gleiche Größe)
  • Die Seitentabelle (Page Table) bildet virtuelle Seiten auf physische Rahmen ab
  • Typische Seitengröße: 4 KiB

Swapping und MMU

  • Swapping: Nicht benötigte Seiten werden auf die Festplatte (Swap-Bereich) ausgelagert
  • Bei Zugriff auf eine ausgelagerte Seite → Page Fault → Kernel lädt Seite zurück
  • Die MMU ist eine Hardware-Komponente der CPU, die die Adressübersetzung in Echtzeit durchführt
  • Ein TLB (Translation Lookaside Buffer) beschleunigt die Adressübersetzung als Cache

Vorteil von Paging: Keine externe Fragmentierung; Prozesse können größer als der physische Speicher sein.

Hierarchisches Dateisystem

  • Dateien sind in einer Baumstruktur (hierarchisch) organisiert
  • Jeder Eintrag ist entweder eine Datei oder ein Verzeichnis (Ordner)
  • Der oberste Knoten heißt Wurzelverzeichnis (Root)
  • Beispiele: / (Linux/macOS), C:\ (Windows)
  • Ein Pfad beschreibt den Weg durch den Baum (z. B. /home/user/dokument.txt)

Mount Points und Verzeichnisstruktur

  • Mount Points: Einbinden eines Dateisystems in die Verzeichnisstruktur
  • Ein USB-Stick wird z. B. unter /media/usb oder /mnt eingehängt
  • Die FHS (Filesystem Hierarchy Standard) definiert die Verzeichnisstruktur unter Linux
  • Wichtige Verzeichnisse: /bin, /etc, /var, /home, /tmp
  • Metadaten werden in Inodes gespeichert (Besitzer, Berechtigungen, Zeitstempel, Blockadressen)

Grundlagen der Virtualisierung

  • Virtual Machine Monitor (VMM) / Hypervisor – Software, die mehrere Gast-Betriebssysteme auf einer physischen Maschine ausführt
  • Typ-1-Hypervisor (bare-metal): Läuft direkt auf der Hardware (z. B. VMware ESXi, Hyper-V, KVM)
  • Typ-2-Hypervisor (gehostet): Läuft auf einem Host-Betriebssystem (z. B. VirtualBox, VMware Workstation)
  • Container (z. B. Docker): Teilen sich den Host-Kernel, isolieren nur den User-Space → schlanker als VMs

Virtualisierung ermöglicht: Konsolidierung, Isolation, Snapshots, Live-Migration, Testumgebungen.
