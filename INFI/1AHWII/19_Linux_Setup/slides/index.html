INFI - 1AHWII

Linux-Geschichte und Philosophie (1)

  • Linus Torvalds entwickelte 1991 den Linux-Kernel
  • Kombination mit dem GNU-Projekt (Richard Stallman, 1983)
  • GNU = "GNU's Not Unix" – freie Unix-kompatible Software
  • Linux ist ein freies, Open-Source-Betriebssystem
  • Lizenz: GNU GPL (General Public License)

Linux-Geschichte und Philosophie (2)

  • Open Source = Quellcode ist öffentlich einsehbar
  • Jeder kann den Code verbessern, anpassen und weitergeben
  • Linux läuft auf Servern, Embedded-Geräten, Android und Desktops
  • Weltweit dominierendes Server-Betriebssystem (> 70 %)
  • Grundprinzip: "Everything is a file"

Linux-Distributionen (2)

Paketmanager verwalten Installation, Update und Deinstallation von Software

  • Debian/Ubuntu: apt (Advanced Package Tool) – .deb-Pakete
  • Fedora/RHEL: dnf (Nachfolger von yum) – .rpm-Pakete
  • Arch Linux: pacman
  • Ein Distributionswechsel bedeutet oft auch einen anderen Paketmanager

VirtualBox-Setup (1) – VM erstellen

  1. Oracle VirtualBox installieren (virtualbox.org)
  2. Neue VM anlegen: Maschine → Neu
  3. Name: Ubuntu-VM, Typ: Linux, Version: Ubuntu (64-bit)
  4. Arbeitsspeicher: 2048 MB (2 GB)
  5. Festplatte: 20 GB, dynamisch alloziert

VirtualBox-Setup (2) – Ubuntu installieren

  1. Ubuntu-ISO herunterladen von ubuntu.com
  2. ISO in das virtuelle Laufwerk einlegen:
    VM → Einstellungen → Massenspeicher → CD hinzufügen
  3. VM starten und Installation durchführen
  4. Sprache, Tastaturlayout (Deutsch), Benutzer anlegen

VirtualBox-Setup (3) – Erster Start

  • Nach der Installation: Login mit Benutzername und Passwort
  • Ubuntu-Desktop erkunden (GNOME-Desktop)
  • Terminal öffnen: Strg + Alt + T
  • Oder: Aktivitäten → Terminal
  • Gast-Erweiterungen installieren für bessere Integration

Erste Schritte im Terminal

Der Shell-Prompt zeigt: benutzer@rechner:~$

whoami    # Wer bin ich?
pwd       # Wo bin ich? (present working directory)
clear     # Bildschirm leeren
echo "Hallo Welt"  # Text ausgeben
  • whoami – aktueller Benutzername
  • pwd – aktuelles Verzeichnis (print working directory)
  • clear – Terminal leeren (oder Strg + L)

Wichtige Linux-Befehle (1)

ls          # Dateien und Ordner auflisten
ls -l       # Detaillierte Ansicht
ls -a       # Auch versteckte Dateien zeigen

cd /etc     # In Verzeichnis wechseln
cd ..       # Ein Verzeichnis nach oben
cd ~        # Ins Home-Verzeichnis

mkdir neuer_ordner    # Ordner erstellen
mkdir -p a/b/c        # Verschachtelte Ordner

Wichtige Linux-Befehle (2)

cp datei.txt kopie.txt          # Datei kopieren
cp -r ordner/ ordner_backup/    # Ordner kopieren

mv datei.txt zielordner/        # Verschieben
mv alt.txt neu.txt              # Umbenennen

rm datei.txt                    # Datei löschen
rm -r ordner/                   # Ordner löschen
rm -rf ordner/                  # Ohne Nachfrage (Vorsicht!)

touch neue_datei.txt            # Leere Datei erstellen
cat datei.txt                   # Dateiinhalt anzeigen

Dateirechte (1) – rwx

Jede Datei hat drei Kategorien von Rechten:

KategorieBedeutung
Besitzer (user)Der Eigentümer der Datei
Gruppe (group)Eine Benutzergruppe
Andere (others)Alle übrigen

Rechte: r (read = 4), w (write = 2), x (execute = 1)

-rwxr-xr--  1 benutzer gruppe  1024 datei.txt
# Besitzer: rwx (7), Gruppe: r-x (5), Andere: r-- (4)

Dateirechte (2) – chmod, chown

chmod 755 datei.sh     # Rechte auf rwxr-xr-x setzen
chmod u+x datei.sh     # Ausführrecht für Besitzer hinzufügen
chmod g-w datei.txt    # Schreibrecht für Gruppe entfernen

chown benutzer datei.txt          # Besitzer ändern
chown benutzer:gruppe datei.txt   # Besitzer und Gruppe ändern
chown -R benutzer:gruppe ordner/  # Rekursiv für Ordner

chown kann nur root ausführen

Zusammenfassung

  • Linux: Open-Source-Betriebssystem von Linus Torvalds (1991)
  • Distributionen: Ubuntu, Debian, Fedora – jeder mit eigenem Paketmanager
  • VirtualBox ermöglicht Linux-Installation auf jedem Rechner
  • Wichtige Befehle: ls, cd, mkdir, cp, mv, rm, touch, cat
  • Dateisystem: hierarchisch, beginnt bei /
  • Dateirechte: rwx für Besitzer, Gruppe, Andere – änderbar mit chmod/chown
