Ein-/Ausgabe-Ports im Überblick

  • Die CPU kommuniziert mit Peripherie über Ports
  • Ein Port ist eine nummrierte Adresse im I/O-Adressraum
  • I/O-Ports sind getrennt vom Arbeitsspeicher (IN/OUT-Befehle)
  • Beispiele: Parallelport (378h), Tastatur (60h), Timer (40h)
Port-AdresseGerät
60hTastatur-Controller
378h – 37FhParallelport (LPT1)
3F8h – 3FFhSerielle Schnittstelle (COM1)

Die IN/OUT-Befehle

  • IN – liest ein Byte/Word von einem Port
  • OUT – schreibt ein Byte/Word zu einem Port
; Beispiel: Wert von Port 60h (Tastatur) lesen
IN  AL, 60h     ; AL = Byte von Port 60h

; Beispiel: Wert zu Port 378h (LPT1) senden
MOV AL, 0FFh    ; Alle Bits auf 1 setzen
OUT 378h, AL    ; Ausgabe an Port 378h

; Beispiel mit DX (16-Bit-Port-Adresse)
MOV DX, 378h
OUT DX, AL
  • Für Ports < 256 kann die Adresse direkt angegeben werden
  • Für Ports > 256 muss die Adresse in DX stehen

Die Ampelschaltung als Zustandsautomat

  • Eine Ampel durchläuft mehrere Zustände in einer festen Reihenfolge
  • Jeder Zustand hat eine definierte Dauer (Timing)

Standard-Ampelphasen:

ZustandFarbeDauer
1Rotca. 30 s
2Rot-Gelbca. 3 s
3Grünca. 25 s
4Gelbca. 3 s
  • Die Ampel wechselt zyklisch: 1 ⇒ 2 ⇒ 3 ⇒ 4 ⇒ 1

Timing und Verzögerungsschleifen

  • Die CPU arbeitet viel schneller als die Ampel schalten muss
  • Wir brauchen Verzögerungsschleifen (Delay-Loops)
; Einfache Verzögerungsschleife
delay:
    MOV CX, 0FFFFh    ; Zähler maximal
delay_loop:
    NOP               ; Leerbefehl (keine Operation)
    LOOP delay_loop   ; CX--, springe wenn CX != 0
    RET
  • Die Dauer hängt von der Taktfrequenz der CPU ab
  • LOOP dekrementiert CX und springt, wenn CX ≠ 0

Verschachtelte Verzögerungsschleifen

  • Eine einzelne Schleife reicht oft nicht für lange Verzögerungen
  • Lösung: verschachtelte Schleifen (äußerer und innerer Zähler)
; Verzögerung mit zwei Schleifen (ca. 1 s bei 4,77 MHz)
delay_long:
    MOV BX, 10         ; Äußerer Zähler
outer_loop:
    MOV CX, 0FFFFh     ; Innerer Zähler
inner_loop:
    NOP
    LOOP inner_loop
    DEC BX
    JNZ outer_loop     ; BX--, springe wenn BX != 0
    RET
  • Äußerer Zähler (BX) steuert die Anzahl der Durchläufe
  • Innerer Zähler (CX) erzeugt die Basisverzögerung

Programmstruktur – Übersicht

  • Das Ampel-Programm hat folgende Struktur:
ORG 100h

.DATA
    ; Konstanten für Zustände und Verzögerungen

.CODE
start:
    ; Initialisierung
main_loop:
    ; Zustand ausgeben (OUT)
    ; Verzögerung
    ; Nächster Zustand
    JMP main_loop

; Hilfsfunktionen
delay:
    ; Verzögerungsschleife
    RET
  • Die Hauptschleife läuft unendlich (Endlos-Schleife)
  • Hilfsfunktionen wie delay werden mit CALL aufgerufen

Testen der Ampelschaltung

  • Zum Testen wird ein Emulator oder eine Hardware-Simulation verwendet
  • Mögliche Testumgebungen:
    • DOSBox mit TASM/MASM
    • EMU8086 (grafischer Emulator mit Port-Simulation)
    • Bochs/QEMU (vollständige PC-Emulation)

Test-Schritte:

  1. Assemblieren (übersetzen) des Quelltextes
  2. Linken zur ausführbaren Datei (EXE/COM)
  3. Ausführen im Emulator – Port-Ausgabe beobachten
  4. Timing überprüfen und Zähler ggf. anpassen

Debuggen des Ampel-Programms

  • Häufige Fehler beim Assembler-Projekt:
    • Falsche Port-Adresse – keine Ausgabe sichtbar
    • Endlosschleife ohne Ausgabe – INIT vergessen
    • Falsche Bit-Muster – falsche Lampen leuchten
    • Timing zu kurz/lang – Zähler falsch gewählt

Debugging-Methoden:

  • Step-by-Step durch das Programm gehen
  • Register (AX, DX, CX) nach jedem Schritt prüfen
  • Breakpoints vor jedem OUT-Befehl setzen
  • Mit kleinen Zählerwerten testen, dann vergrößern
