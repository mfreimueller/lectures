13 - BIOS & UEFI

INFI - 1AHWII

BIOS – Grundlagen

  • BIOS = Basic Input/Output System
  • Firmware auf dem Mainboard – der erste Code, der beim Einschalten läuft
  • Aufgaben:
    • Hardware initialisieren und testen (POST)
    • Bootloader von einem Speichermedium laden
    • Einfache Treiber für Tastatur, Maus, Festplatten bereitstellen
    • BIOS-Setup (Konfiguration speichern)
  • Gespeichert in einem ROM- oder Flash-Chip auf dem Mainboard

POST und CMOS

  • POST (Power-On Self-Test):
    • Überprüft CPU, RAM, Grafikkarte, Tastatur, Massenspeicher
    • Signalisiert Fehler durch Pieptöne (Beep-Codes)
    • Wichtigster Test: RAM-Initialisierung und Grafikkarten-Erkennung
  • CMOS (Complementary Metal-Oxide-Semiconductor):
    • Kleiner Speicher für BIOS-Einstellungen (Bootreihenfolge, Datum/Zeit)
    • Wird durch eine CMOS-Batterie (Knopfzelle, CR2032) gepuffert
    • Bei leerer Batterie: Einstellungen gehen verloren, Datum/Zeit zurückgesetzt

BIOS-Speicher und Flash-Update

  • Der BIOS-Chip ist ein Flash-EEPROM:
    • Nicht flüchtig (Daten bleiben ohne Strom erhalten)
    • Kann durch ein BIOS-Update (Flashing) aktualisiert werden
    • Größe typischerweise 8 MB – 32 MB (viel kleiner als ein OS)
  • BIOS-Update:
    • Behält Sicherheitslücken oder fügt CPU-Unterstützung hinzu
    • Risiko: Stromausfall während des Flashens = defektes Mainboard
    • Viele Mainboards haben einen Dual-BIOS (Reserve-Chip)
  • CMOS löschen (Reset): Batterie entfernen oder Jumper setzen

Der Bootvorgang – Schritt 1 und 2

  • Schritt 1: Einschalten (Power-On)
    • Netzteil versorgt Mainboard mit Standby-Spannung (+5VSB)
    • Power-Button wird gedrückt → Netzteil startet alle Spannungen
    • CPU erhält Reset-Signal und startet an der Reset-Vektoradresse
  • Schritt 2: POST und Hardware-Initialisierung
    • BIOS führt den POST durch (CPU, RAM, Grafik, Tastatur)
    • Grafikkarte wird initialisiert → erstes Bild erscheint
    • Speichertimer werden konfiguriert, PCI-Geräte werden erkannt
    • Bei Fehlern: Pieptöne oder Fehlermeldung auf dem Bildschirm

Der Bootvorgang – Schritt 3 bis 5

  • Schritt 3: Bootloader laden
    • BIOS durchsucht die Bootreihenfolge (z. B. USB → SSD → DVD)
    • Liest den ersten Sektor (MBR/GPT) des gewählten Mediums
    • Bootloader wird in den RAM geladen und ausgeführt (z. B. GRUB, Windows Boot Manager)
  • Schritt 4: Betriebssystem-Kernel starten
    • Bootloader lädt den OS-Kernel (z. B. vmlinuz bei Linux, ntoskrnl.exe bei Windows)
    • Kernel übernimmt die Kontrolle über die Hardware
  • Schritt 5: Benutzeranmeldung
    • Systemdienste und Treiber werden geladen
    • Graphische Oberfläche (GUI) startet
    • Benutzer meldet sich an

UEFI – Vorteile im Detail

  • Grafische Oberfläche: Mausbedienung, intuitiver als BIOS
  • Größere Festplatten: GPT erlaubt Partitionen > 2 TB und mehr als 4 primäre Partitionen
  • Schnelleres Booten: UEFI kann den Bootvorgang optimieren
    • Kein Wechsel in den 16-Bit-Real-Mode
    • Parallelisierung der Initialisierung
  • Modularer Aufbau: EFI-Treiber können nachgeladen werden (z. B. Netzwerk-Stack)
  • Netzwerk-Boot: UEFI unterstützt PXE-Boot ohne zusätzliche ROMs
  • Kompatibilität: CSM (Compatibility Support Module) für Legacy-Betrieb

Secure Boot

  • Secure Boot ist eine UEFI-Funktion, die das Ausführen von nicht signiertem Code verhindert
  • Funktionsweise:
    • Nur Bootloader mit einer gültigen digitalen Signatur dürfen ausgeführt werden
    • Die Signaturen werden mit einem Schlüssel verglichen (PK – Platform Key)
    • Schützt vor Bootkits und Rootkits, die vor dem OS laden
  • Voraussetzung für Windows 11 – muss aktiviert sein
  • Kann im UEFI-Setup deaktiviert werden (z. B. für Linux-Distributionen ohne Signatur)
  • Kritik: Herstellerkontrolle über die Hardware (Microsoft-Schlüssel)

GPT vs. MBR Partitionierung

  • MBR (Master Boot Record):
    • Erster Sektor (512 Byte) eines Datenträgers
    • Enthält: Bootloader-Code + Partitionstabelle (4 Einträge à 16 Byte)
    • Maximale Größe: 2 TB (32-Bit-Adressierung, 512-Byte-Sektoren)
    • Maximal 4 primäre Partitionen (oder 3 primäre + erweiterte Partition)
  • GPT (GUID Partition Table):
    • Teil des UEFI-Standards
    • Maximale Größe: ~9,4 Zettabyte (64-Bit-Adressierung)
    • Bis zu 128 Partitionen unter Windows (unbegrenzt theoretisch)
    • Redundante Partitionstabelle am Ende der Festplatte → Fehlertoleranz

BIOS-Setup und Konfiguration

  • Zugriff: Entf (DEL), F2, F10 oder Esc beim Start
  • Wichtige Einstellungen:
    • Bootreihenfolge: Welches Medium zuerst booten (USB, DVD, SSD)
    • Secure Boot: Aktivieren/Deaktivieren
    • Boot Mode: Legacy (CSM) oder UEFI, ggf. Both
    • SATA-Modus: AHCI (empfohlen) vs. IDE (Kompatibilität) vs. RAID
    • XMP/DOCP: RAM-Übertakten (höhere Frequenz freischalten)
  • Vorsicht: Falsche Einstellungen können das System instabil machen
  • Reset auf Standardwerte (Load Optimized Defaults) behebt meistens Probleme

Zusammenfassung

  • BIOS ist die Firmware, die Hardware initialisiert und den Bootvorgang startet
  • POST testet die Hardware, BIOS-Einstellungen werden im CMOS gespeichert
  • Bootvorgang: Einschalten → POST → Bootloader → OS-Kernel → Anmeldung
  • UEFI ist der moderne Nachfolger von Legacy-BIOS mit grafischer Oberfläche, GPT-Support und Secure Boot
  • GPT erlaubt größere Festplatten (> 2 TB) und mehr Partitionen als MBR
  • Secure Boot schützt vor nicht signierten Bootloadern
  • BIOS-Setup: Bootreihenfolge, Secure Boot, Boot-Modus, XMP
